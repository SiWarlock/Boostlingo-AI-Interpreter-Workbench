## Decisions made
- **Real provider = thin manual-smoke transport shell + a pure unit-TDD'd mapper** (lessons §18/§20/§21). Callback SDKs (Deepgram) bridge to `IAsyncEnumerable` via `Channel<T>`; raw-HTTP providers (OpenAI) are TDD'd end-to-end via a mock `HttpMessageHandler` returning a canned SSE / binary stream.
- **SDK surfaces verified by reflection/source, not docs.** Caught: Deepgram v6.6.1 `PreRecordedSchema`/`LiveSchema` namespaces + the no-`HttpStatusCode`-on-exception reality (C.6); OpenAI Responses API nests `reasoning.effort`/`text.verbosity` + has no `[DONE]` (C.2); `gpt-4o-mini-tts` defaults `stream_format` to `sse` — must send `"audio"` for raw chunks (C.3).
- **C.6 (pulled forward by the user):** recover the HTTP status from the Deepgram exception's semantic `err_code` string (`TOO_MANY_REQUESTS`→429, `INVALID_AUTH`→401, `INSUFFICIENT_PERMISSIONS`→403, `Bad Request`→400) → vendor-agnostic `MapStatus`; 5xx-with-body unrecoverable → `stt.unknown` (accepted residual).
- **OCE-filter polarity** (§20): `catch (OCE) when (ct.IsCancellationRequested){throw;}` + generic-arm-maps-timeout — NOT inverted (inverting with a trailing generic catch would swallow caller-cancel). Per-stage timeout covers the whole streamed read loop.
- **C.4a:** the orchestrator stays unchanged except the `turn.completed`/`translation.final` stamp edits; the boundary (not the orchestrator) owns the encoding allowlist; single-sender WS design (pump receives + stashes `recording.stopped`; main loop is the sole sender); audio is streamed but never accumulated/persisted (invariant #3).

### Limitation / Finding (→ G.5 comparison write-up + ARCH-014 note)
- **TTS cost in audio-streaming mode:** OpenAI reports no usage block with `stream_format:"audio"`, so the cascade cost uses a **target-character proxy** for TTS (documented assumption) and degrades cleanly when unpriceable. Translation tokens are surfaced via the FORK-1a Metadata; STT from recording duration. Precise TTS cost would require `sse`-mode usage, which complicates the raw-byte reader — judged not worth it. A real cost-fidelity limitation to disclose in the comparison write-up.

## How to use what was built
- **Run a real cascade turn:** configure `DEEPGRAM_API_KEY` + `OPENAI_API_KEY` (real providers auto-selected by key-presence; set `USE_FAKE_PROVIDERS=true` for a keyless local run); create a session + turn via the B.9 HTTP endpoints; open `WS /api/cascade/stream`; send the `start` frame (the backend-minted `turnId`), then binary linear16 PCM frames, then `{type:"stop"}`. The server streams `transcript`/`latency`/`audio`/`cost`/`done`; the turn persists.
