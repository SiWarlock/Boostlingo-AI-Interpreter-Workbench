## Decisions made

- **C.4b 2-commit packaging (not 3).** The bounded channel + Origin both live in `CascadeWebSocketEndpoint.cs`; whole-file staging (no interactive `git add -p`) can't split them, and the endpoint depends on the robustness helpers — so robustness committed first, the SECURITY surface (bounded channel + Origin, the commit tip) second. Isolates the security surface in one reviewable commit without a fragile mid-file intermediate. (Orchestrator + lead approved.)
- **C.4b STT no-final (Q4): pending-partial = lost final.** A started translation/TTS stage without its terminal → `<stage>.unknown` Failed; STT with dangling partials but no final → `stt.unknown`; STT clean end with no partial/final (silence) → `Completed`. Honest fail-closed without flagging valid silence as an error.
- **C.4b new error codes:** `cascade.invalid_request` (malformed/missing-field/oversized start frame) + `cascade.forbidden_origin` (403) — honest over reusing `cascade.invalid_audio` (which stays the SECURITY encoding-allowlist code).
- **C.5 reuse the streaming orchestrator** (Q1) — the blob adapter feeds a 1-frame container-encoding stream into the unchanged pipeline (pre-recorded STT routing already exists in `DeepgramSttProvider`). Maximal reuse; the per-segment + fail-closed + cost glue all come free (ARCH-008/011).
- **C.5 blob STT cost: degrade, don't proxy** (Q4). The pre-recorded path carries no in-band audio-duration signal in the `SttEvent` contract; a processing wall-clock would be a ~30× undercount. Passing STT minutes unsupplied → the composite cost degrades to **unavailable** for blob turns (honest; no synthetic metric — streaming-honesty / lesson §9). The streaming WS path prices fully.
- **C.5 response carries target audio base64 in-body** (Q2) — the non-streaming path must deliver audio the client can play; it's in the response only, **never persisted** (invariant #3; `AssembleTurn` has no Audio case).
- **C.5 strip MIME params before the upload allowlist** (review catch) — a browser `MediaRecorder` sends `audio/webm; codecs=opus`; an exact-match lookup would 415 the primary Phase-D capture path.
- **C.5 bound the multipart body** (review catch) — `FormOptions`/Kestrel to the cap + 1MB envelope, closing the 128MB-default buffering window; the precise per-file cap + `cascade.invalid_audio` 413 still come from the validator.

## How to use what was built

- **Streaming cascade (primary):** as in session 004 — `WS /api/cascade/stream`, `start` frame + binary `linear16` PCM + `{type:"stop"}`. C.4b adds: the endpoint now validates `Origin` (must match `FRONTEND_ORIGIN`), bounds the audio buffer, and fails closed on a terminal-less provider stream.
- **Blob fallback (non-streaming):** `POST /api/cascade/turn` (multipart) with fields `sessionId`, `turnId` (backend-minted), `source`/`target` (`en`/`es`), `translationModel`, `ttsVoice`, and an `audio` file (`audio/webm|ogg|wav|mpeg|mp4`, ≤ `CASCADE_MAX_UPLOAD_BYTES`). Returns the assembled turn + the target audio as base64. Create the session + turn via the B.9 HTTP endpoints first. Keyless local run: `USE_FAKE_PROVIDERS=true`.
