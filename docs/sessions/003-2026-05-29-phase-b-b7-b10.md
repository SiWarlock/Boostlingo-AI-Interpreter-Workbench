## Decisions made

- **B.9 split a/b/c-i/c-ii** — B.9 mixed two safety invariants (#4 global handler, #1 config) + a large controller; safety-isolation requires separate commits, and session-CRUD vs turn-lifecycle are separable. Four commits, each reviewable.
- **`UiError` server-only `[JsonIgnore] HttpStatusCode`** — body stays an exact TS mirror; the handler sets the response status line from it.
- **Controller collaborators get a thin interface** (`IConfigService`/`ISessionService`) as the test seam (enables the `ConfigureTestServices` boundary proof); other concrete services keep no interface.
- **`AddControllers().AddJsonOptions(JsonDefaults.Apply)`** — MVC's JsonOptions is separate from the minimal-API one; needed for the enum-bearing DTOs.
- **`/end` MUST-persist, `/complete` best-effort** (ARCH-016) — both return 200 with a `persistenceWarning` `UiError` on write failure, never 500 (the in-memory end/complete still happened).
- **`PersistedPath` is filename-only** (no absolute-path/username disclosure; still honors Flow F).
- **Boundary input caps via DataAnnotations on record PARAMETERS** (not `[property:]` — MVC throws otherwise); oversized → framework 400 ProblemDetails (consistent with the unrouted-404/405 decision).
- **`/complete` Status coerced to terminal** (`Failed` if reported, else `Completed`) — a client can't drag a turn backwards.
- **`turnId`/`sessionId` server-generated in the store**; client `{id}`/`{turnId}` are in-memory-lookup-only, never reach the FS.
- **B.10 scoped tight (6)** — net-new error-preservation trio + interface-level success anchors; cancellation (B.2) + mapper truth table (B.1) referenced, not re-tested.

## How to use what was built

The backend now serves the full session lifecycle over HTTP with fakes: `GET /api/config` (capability gating), `POST /api/sessions` → `POST …/turns` → `…/turns/{id}/events` → `…/turns/{id}/complete` → `POST …/{id}/end` (persists JSON) + `GET …/{id}` / `…/{id}/summary`. Any unhandled error returns a sanitized `UiError`. Phase C swaps fakes → real providers behind the unchanged B.1 interfaces (the B.10 suite is where real-provider conformance lands).
