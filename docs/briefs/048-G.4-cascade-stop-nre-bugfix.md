# /tdd brief ‚Äî cascade_stop_nre_bugfix

> **Real-key-smoke bug-fix (HIGH).** A live cascade run crashes at each turn's Stop with an unhandled `NullReferenceException` ‚Äî the user's "I keep getting failures." Invisible to the fake-tested suite (real-network exempt, ARCH-020); now reproducible. **This IS a `/tdd` slice** ‚Äî the NRE is deterministic; write a failing regression test that reproduces it, then fix.

## Feature
Fix the `NullReferenceException` thrown on the cascade WebSocket **Stop / finalize path** so a turn's Stop never crashes ‚Äî it finalizes cleanly (or fails with a sanitized `ProviderError`/`UiError`, ARCH-018), never an unhandled exception.

## Use case + traceability
- **Task ID:** G.4 (stability hardening ‚Äî smoke-surfaced 2026-05-31)
- **Architecture sections:** `ARCH-011` (cascade streaming + WS finalize), `ARCH-018` (errors sanitized + degrade-don't-crash), `ARCH-016` (persist-on-finalize). The stop path must fail closed + sanitized, never crash.
- **Evidence (live log):** `[Error] Stop: System.NullReferenceException ‚Äî Object reference not set` at **each** turn's Stop on `WS /api/cascade/stream`.

## Diagnosis (from the orchestrator's read ‚Äî confirm/refine via reproduction)
- The `"Stop:"` error log + stop handling live in `Cascade/CascadeWebSocketEndpoint.cs` (stop-frame ‚Üí `onStop()` ~L246; finalize/persist path ~L126‚ÄìL215).
- **The try/catch around the stop/finalize path catches only `OperationCanceledException` (~L62‚Äì71), NOT `NullReferenceException`** ‚Üí an unhandled deref propagates + crashes the turn.
- **Candidate null sites (pin the real one via reproduction):**
  1. `PersistAsync` session read `store.Get(p.SessionId)` (~L215) returning **null** ‚Üí deref on `session.Turns`/etc. A **session-eviction race** (E.5-backend stale-flush, or `/complete`, landing concurrently with the WS Stop finalize) is the leading hypothesis.
  2. `recordingStopped` (~L126/L162/L165) **null** on a stop-before-start / immediate-timeout / double-stop ‚Üí a downstream deref (the `?.Timestamp ?? clock.UtcNow` at L165 is guarded; look for an unguarded sibling).
  3. `FinalizeTurn` returning `Applied=false` (already-terminal turn from a racing `/complete`) leaving a half-path that derefs.
