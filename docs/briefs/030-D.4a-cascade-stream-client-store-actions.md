# /tdd brief — cascade_stream_client_and_streaming_store_actions

## Feature
The cascade streaming WebSocket client (`cascadeStreamClient`) — opens `WS /api/cascade/stream`, sends the `start` frame → binary PCM frames → `stop`, and dispatches the server messages (`transcript`/`latency`/`audio`/`cost`/`error`/`done`) into the store — plus the **streaming-event store actions** deferred from D.1 (begin/append-transcript/append-latency/cost/fail/complete a turn). This is **D.4a** of the split D.4 (D.4b = `RecordingControls` + the full recording wiring follows).

## Use case + traceability
- **Task ID:** D.4 (slice a — split per the C.4a/C.4b precedent: transport+store first, the orchestrating component second)
- **Architecture sections it implements:** `ARCH-009` (the cascade WS wire contract — client `start`/`stop`, server `transcript`/`latency`/`audio`/`cost`/`error`/`done`), `ARCH-011` (streaming pipeline — **source partials render as they arrive, replaced by finals**; streaming honesty), `ARCH-007` (clean separation — the transport owns wire detail; it dispatches into the store; components render only from the store).
- **Related context:** Builds on D.1 (store, `domain.ts` types incl. the wire `transcript`/`latency`/`cost`/`done` shapes + `TurnViewModel`), D.3 (the capture controller's `onFrame` hands the binary frames this client sends — wired in D.4b). The backend WS surface is COMPLETE + hardened (C.4a/C.4b, `WS /api/cascade/stream`). **D.4a is foundation-ahead-of-callers** — `RecordingControls` (D.4b) drives `.start()/.sendFrame()/.stop()`; closed in the very next slice + the Phase-D-exit audit.

### Verified WS wire contract (from ARCH-009 / Appendix A / the C.4a backend)
- **Client → server:** first text frame `{ type:'start', sessionId, turnId, direction:{source,target}, encoding:'linear16', sampleRate, translationModel, ttsVoice }`; then **binary** linear16 PCM frames; final text frame `{ type:'stop' }`.
- **Server → client (text frames):** `{type:'transcript', segment:{...TranscriptSegment}}` · `{type:'latency', event:{...LatencyEvent}}` · `{type:'audio', contentType, seq, base64}` · `{type:'cost', estimate:{...CostEstimate}}` · `{type:'error', error:{...ProviderError}}` · `{type:'done', turnId, status}`. (Binary audio frames are an allowed optimization — but the backend sends base64 `audio` text frames by default.)
- **Origin:** the WS upgrade is `Origin`-validated server-side (C.4b) against `FRONTEND_ORIGIN` — so the browser-presented origin must match (true for both the dev proxy + direct; D.1's note).

## TDD scope
**Test-first (deterministic):**
- `toWebSocketUrl(apiBase, location)` — `''` → `ws(s)://<location.host>` (proxy); `http://…` → `ws://…`; `https://…` → `wss://…`. Pure (location injected).
- `buildStartFrame(params)` — the `start` message object. Pure.
- `dispatchCascadeMessage(rawText, { store, onAudio })` — parse a server text frame → the right store action / `onAudio` callback; unknown/`malformed` type → ignored-safely (no throw). Pure dispatch router. **The meat.**
- The **store streaming actions** (each deterministic): `beginTurn`, `appendTranscriptSegment` (partial-replace/finalize), `appendLatencyEvent`, `setTurnCost`, `failTurn`, `completeTurn`.
- The **client lifecycle** via an **injected fake WebSocket** (open → send start → frames → stop → dispatch inbound → done/close): assert the start frame is sent on open, frames forward as binary, stop is sent, inbound messages dispatch, and an abnormal close surfaces a `UiError`.

**Exempt (manual-smoke):** the real browser `WebSocket` network + binary transmission, and the live backend round-trip (the C/D demo-checklist). The client wraps an injectable `WebSocket` so its orchestration IS unit-tested against a fake.

## Acceptance criteria
- [ ] `cascadeStreamClient` (`web/src/cascade/cascadeStreamClient.ts`): `createCascadeStreamClient({ store, onAudio, wsFactory? })` → `{ start(params), sendFrame(buf), stop(), close() }`. On `start`: open the WS (binaryType `'arraybuffer'`), send `buildStartFrame` on open; `sendFrame` forwards a binary frame; `stop` sends `{type:'stop'}`; inbound text → `dispatchCascadeMessage`. `wsFactory` defaults to the global `WebSocket` (injected as a fake in tests).
- [ ] `dispatchCascadeMessage` routes: `transcript` → `store.appendTranscriptSegment`; `latency` → `store.appendLatencyEvent`; `audio` → `onAudio(chunk)` (**NOT the store** — raw audio never enters the store/persistence, invariant #3); `cost` → `store.setTurnCost`; `error` → `store.failTurn`; `done` → `store.completeTurn`. Malformed JSON / unknown `type` → ignored (logged, no throw).
- [ ] Store actions (added to `sessionStore.ts`): `beginTurn({turnId,mode,direction})` (set `currentTurn`, `turnStatus:'recording'`); `appendTranscriptSegment(seg)` (route by `role`; **replace the trailing non-final partial, finalize on `isFinal`** — see Step-2.5 Q1); `appendLatencyEvent(evt)` (map into `currentTurn.latency.stages`/top-level); `setTurnCost(estimate)` (`currentTurn.estimatedCostUsd`/`…PerMinute`); `failTurn(uiError)` (`addError` + `currentTurn.status:'failed'`, `turnStatus:'failed'`); `completeTurn(turnId,status)` (finalize `currentTurn` → push to `turns[]`, clear `currentTurn`, `turnStatus:'completed'|'failed'`). Each: new state ref, notifies.
- [ ] An abnormal WS close (a close before `done`, or `onerror`) → `store.failTurn` with a `cascade.connection_lost` `UiError` (see Step-2.5 Q6).
- [ ] Clean separation (ARCH-007): the client imports the store (to dispatch) + the `domain.ts` types — components never import the client; the client owns all wire detail. Raw audio goes only to the `onAudio` callback (→ D.5 playback), never the store.
- [ ] All unit tests in `web/src/cascade/cascadeStreamClient.test.ts` + the new `sessionStore` streaming-action cases pass; `/preflight` clean.

## RED test outline (Step 2)

### cascadeStreamClient (`web/src/cascade/cascadeStreamClient.test.ts`)
1. **`to_websocket_url_derives_scheme`** — `''`+https-location → `wss://host/api/cascade/stream`; `http://localhost:5179` → `ws://localhost:5179/...`; `https://…` → `wss://…`. Why: ARCH-029 proxy+direct; WS needs ws(s).
2. **`build_start_frame_shape`** — Asserts the exact `{type:'start',sessionId,turnId,direction,encoding:'linear16',sampleRate,translationModel,ttsVoice}`. Why: ARCH-009 client contract.
3. **`dispatch_routes_each_server_message`** — a `transcript`/`latency`/`cost`/`error`/`done` frame each → the matching store action (spy on the real store); an `audio` frame → `onAudio` (NOT a store mutation); malformed JSON + unknown type → no throw, no store mutation. Why: ARCH-009 server contract + invariant #3 (audio off the store).
4. **`lifecycle_start_sends_frame_on_open`** — with a fake WS: `.start(params)` opens; on the fake's `onopen`, the start frame is `send`-ed; `binaryType==='arraybuffer'`. Why: open-then-start ordering.
5. **`lifecycle_send_frame_and_stop`** — `.sendFrame(buf)` → `ws.send(buf)` (binary); `.stop()` → `ws.send('{"type":"stop"}')`. Why: client→server frame flow.
6. **`abnormal_close_fails_turn`** — a fake `onclose` (unclean) / `onerror` before `done` → `store.failTurn` with `cascade.connection_lost`; a close AFTER `done` → no error. Why: connection-loss surfacing (Q6).

### sessionStore streaming actions (`web/src/state/sessionStore.test.ts` — added)
7. **`begin_turn_sets_current_turn_recording`** — `beginTurn(...)` → `currentTurn` set (empty transcripts, status recording), `turnStatus:'recording'`, new ref.
8. **`append_transcript_partial_then_final`** — a source partial → `currentTurn.sourceTranscript=[{text,isFinal:false}]`; a second partial **replaces** it (not appends); a final → the trailing entry becomes `{...,isFinal:true}`; a following partial appends a NEW running entry. A target-role segment routes to `targetTranscript`. Why: ARCH-011 partials-replaced-by-finals (Q1).
9. **`append_latency_and_cost`** — `appendLatencyEvent` maps a `{name,stage,relativeMs}` into `currentTurn.latency.stages[name]` (+ top-level for the speech-end metrics); `setTurnCost` sets the cost fields. Why: metrics/cost on the turn (ARCH-013/014).
10. **`fail_turn_and_complete_turn`** — `failTurn(uiError)` → `errors` has it + `currentTurn.status:'failed'`; `completeTurn(turnId,'completed')` → `currentTurn` finalized into `turns[]`, `currentTurn` cleared, `turnStatus:'completed'`. Why: turn finalization.

## Things to flag at Step 2.5
1. **Transcript partial-handling in the store.** Default vote: **replace the trailing non-final partial; finalize on `isFinal`; a partial after a final starts a new running entry** — gives the panel a clean "running partial → finalized history" sequence (ARCH-011). (Alt: store every segment + let D.6 filter — more state, more panel logic. I prefer the store normalizing it.)
2. **Audio message → `onAudio` callback, NOT the store.** Default vote: **callback** — raw audio must never enter the store/persistence (invariant #3); D.5's playback controller consumes `onAudio`. The store may later get a `playback.started` latency stamp (D.5), but never the bytes.
3. **Inject the `WebSocket` factory.** Default vote: **yes** — `wsFactory?` defaulting to the global `WebSocket`, so the client's lifecycle is TDD'd against a fake socket (the frontend analog of the backend's WS-endpoint tests). Only the real network is smoke.
4. **WS URL derivation.** Default vote: `toWebSocketUrl(apiBase, location)` — `''` → derive from `location` (proxy); `http(s)://` → `ws(s)://`. Pure, location injected.
5. **Store-action set + a `cascadeMessages.ts` split?** Default vote: the **6 actions** above; keep `dispatchCascadeMessage` + the pure helpers either in `cascadeStreamClient.ts` or a sibling `cascadeMessages.ts` if it tests cleaner — implementer's call, flag if split.
6. **Abnormal-close handling.** Default vote: a close/error **before `done`** → `failTurn(cascade.connection_lost)`; a close after `done` is normal (no error). Track a `done`-received flag in the client.
