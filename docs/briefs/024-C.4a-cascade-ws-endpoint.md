# /tdd brief — cascade_ws_endpoint (C.4a core)

## Feature
Build the cascade streaming WebSocket endpoint (`WS /api/cascade/stream`) that wires the three **real** providers (C.1/C.2/C.3) into the **unchanged** B.4 `CascadeStreamingOrchestrator` and streams a real cascade turn end-to-end: client `start` → binary PCM frames → `stop`; server emits `transcript`/`latency`/`audio`/`cost`/`error`/`done`; the turn persists. Includes DI swap (fakes↔real by config), the `turn.recording.*` lifecycle events + `LatencyStage.Overall`, cost emission, and the **`encoding` allowlist** (SECURITY — its own commit). **C.4a is the core integration; C.4b carries the remaining WS-boundary hardening** (Origin/ContentType-clamp/double-End/stream-without-terminal).

## Use case + traceability
- **Task ID:** C.4 (this brief = **C.4a**, the core; C.4b = hardening, separate brief).
- **Architecture sections it implements:** `ARCH-009` (§6 Cascade WS protocol — exact message shapes), `ARCH-011` (§8 streaming pipeline — the orchestrator seam), `ARCH-008` (layering — provider logic stays out of the endpoint), `ARCH-019` (boundary validation), `ARCH-013` (turn-lifecycle latency events), `ARCH-005` (`LatencyStage.Overall`).
- **Related context:**
  - **The orchestrator is UNCHANGED (B.4 seam proven).** `CascadeStreamingOrchestrator` already drives STT→translation→TTS over `IAsyncEnumerable<CascadeOutputEvent>` (`Transcript`/`Latency`/`Audio`/`Error`/`Done`); C.4a feeds it real providers + real audio frames and maps its output to WS messages. **Do NOT modify the orchestrator's core loop** (except the `turn.recording.*` stamps + the stream-without-terminal hardening, the latter deferred to C.4b).
  - **Providers are DI-swapped, not rewritten:** the C.1/C.2/C.3 real providers implement the same `ISttProvider`/`ITranslationProvider`/`ITtsProvider` the fakes do; C.4a wires `Program.cs` to bind real-vs-fake by config.
  - **WS protocol is EXACT (ARCH-009):** client `start{sessionId,turnId,direction,encoding,sampleRate,translationModel,ttsVoice}` → binary PCM → `stop{}`; server `transcript{segment}` / `latency{event}` / `audio{contentType,seq,base64}` / `cost{estimate}` / `error{error}` / `done{turnId,status}`.
  - **Reuses:** B.7 `SessionStore` + `SessionPersistenceWriter` (persist the turn, best-effort per-turn per ARCH-016), B.5 `CostEstimator.EstimateCascadeTurn` (cost), B.3 `LatencyEventFactory` (stamps). Lesson §18/§20 (shell + pure-mapper; `Channel<T>` bridge).
  - **TDD posture:** the **WS transport shell** (WebSocket accept, frame read/write loop, PCM→`AudioFrame` `Channel` bridge) is **manual-smoke** (network boundary, like the provider shells). TDD the **deterministic** pieces: the start-frame parse + validation (incl. the encoding allowlist), the `CascadeOutputEvent`→WS-message mapping, the cost computation+emission, and the turn assembly→persist.

## Acceptance criteria (what "done" means)
- [ ] `WS /api/cascade/stream` accepts the ARCH-009 protocol; a real cascade turn streams **source partials → target tokens → first TTS audio all before `done`** (the ARCH-011 streaming contract), and the turn persists (best-effort per-turn).
- [ ] **DI swap:** `Program.cs` binds real providers when configured, fakes otherwise (selector rule settled at Step 2.5); the orchestrator + provider interfaces are unchanged.
- [ ] **`CascadeOutputEvent`→WS-message mapping (pure, unit-tested):** `Transcript→{type:transcript,segment}`, `Latency→{type:latency,event}`, `Audio→{type:audio,contentType,seq,base64}`, `Error→{type:error,error:ProviderError}`, `Done→{type:done,turnId,status}`; camelCase via `JsonDefaults`.
- [ ] **Start-frame parse + validation (pure, unit-tested):** deserialize `start`; **`encoding` allowlist** — a closed set (`linear16`/`pcm`) validated **before** building `CascadeStartParams` (the orchestrator interpolates `Encoding` into `audio/{encoding}` — unvalidated = header-injection at the real provider, ARCH-019); reject invalid → `cascade.invalid_audio`. **This validation is its OWN commit (SECURITY).**
- [ ] **Cost emission:** after the turn, compute `CostEstimator.EstimateCascadeTurn(...)` from the turn's usage (STT audio-seconds, translation tokens) and emit `{type:cost,estimate}` before `done` (degrade to no-cost-message if unavailable, never crash — B.5 `Result`).
- [ ] **`turn.recording.*` + `LatencyStage.Overall`:** emit `turn.recording.started` (on `start`), `turn.recording.stopped` (on `stop`), `turn.completed` (on `done`), stamped **`LatencyStage.Overall`** (new enum member). **Cross-doc:** `LatencyStage.Overall` → ARCH-005 + Appendix A (orchestrator-written hot).
- [ ] **Persist:** on `done`, assemble the `InterpretationTurn` (transcripts/latency/cost/status) → `SessionStore.UpdateTurn` → best-effort persist (per ARCH-016; a write failure does not crash the turn).
- [ ] **No leak:** no provider key / ephemeral secret / raw audio in any WS message or persisted JSON (the existing sentinel + sanitizer hold; provider errors cross as sanitized `ProviderError`).
- [ ] All unit tests pass; full suite green (193 + new); `/preflight` clean.

## RED test outline (Step 2 — `AiInterpreter.Tests/CascadeWebSocketTests.cs`)
TDD the deterministic pieces (the WS transport itself is manual-smoke):

1. **`start_frame_parses_to_cascade_params`** — a valid `start` JSON → the right `CascadeStartParams` (direction, sampleRate, models, voice). Why: ARCH-009 protocol.
2. **`encoding_allowlist_rejects_invalid`** `[Theory]` — `encoding` ∉ {`linear16`,`pcm`} (e.g. `"mp3; x-inject"`) → rejected with `cascade.invalid_audio`, `CascadeStartParams` NOT built. Why: ARCH-019 header-injection guard. **(This + the start-parse are the SECURITY commit.)**
3. **`output_event_maps_to_ws_message`** `[Theory]` — each `CascadeOutputEvent` variant → the exact ARCH-009 server message JSON (`transcript`/`latency`/`audio`(base64+seq+contentType)/`error`/`done`). Why: ARCH-009 message shapes; camelCase.
4. **`cost_computed_and_emitted_before_done`** — given a turn's usage, a `cost` message with a `CostEstimate` is emitted; unavailable pricing → no cost message, no crash. Why: ARCH-014/B.5 degrade.
5. **`turn_lifecycle_events_stamped_overall`** — `turn.recording.started/.stopped`/`turn.completed` are stamped `LatencyStage.Overall`. Why: ARCH-013/ARCH-005.
6. **`turn_assembled_and_persisted_best_effort`** — on `done`, the `InterpretationTurn` carries the accumulated transcripts/latency/cost/status and is persisted; a write failure does not throw. Why: ARCH-016 write tiers.
7. **`provider_error_crosses_as_sanitized`** — an orchestrator `Error(ProviderError)` → `{type:error,error}` with no `ex.Message`/key/stack. Why: invariant #4.

Scope the WS-transport integration (real socket, real providers) to **manual smoke** — do not attempt a deterministic full-socket test. Don't re-test the orchestrator (B.4) or the providers (C.1/2/3).

## Things to flag at Step 2.5
1. **DI swap selector (real↔fake).** By **key presence** (Deepgram + OpenAI keys configured → real; else fake) vs an explicit env toggle (`USE_FAKE_PROVIDERS`). **Default vote: key-presence** (reuses the B.9b `ConfigService` capability logic — a mode is "real" iff its keys are present; consistent with config-gating) **+ an explicit override env for local keyless runs.** Confirm the exact rule + where it's registered.
2. **Pure-mapping seam.** `internal static CascadeWsMapping` (start-parse+allowlist, event→message, DTOs) + the thin endpoint shell — matches §18/§20. **Default vote: yes.**
3. **PCM→`AudioFrame` bridge.** Binary WS frames → `Channel<AudioFrame>` exposed as the orchestrator's `SttRequest.AudioFrames` `IAsyncEnumerable`; `stop` completes the channel. **Default vote: `Channel<AudioFrame>`** (the §18 bridge pattern; cancellation/`stop` completes the writer).
4. **Cost timing + usage source.** Compute `EstimateCascadeTurn` after `stop`/before `done`, from accumulated usage (STT audio-seconds from the recording duration; translation tokens from `TranslationFinal`). **Default vote: post-stop, pre-done; degrade to no-cost on missing usage.** Confirm where translation tokens are captured (the orchestrator's `Done`/an accumulated field).
5. **Commit split within C.4a.** **Default vote: ≥2 commits — (1) the `encoding` allowlist + start-frame validation (SECURITY, isolated), (2) the endpoint + mapping + DI + cost + persist + lifecycle.** You may split (2) further (e.g. pure mapping/DTOs vs transport+DI+persist) if it aids review — your call. The encoding-allowlist-as-own-commit is non-negotiable (safety isolation).
6. **`turnId` ownership.** The `start` frame carries a `turnId` (client echoes the backend-minted one from `POST …/turns`) vs the WS mints it. **Default vote: client sends the backend-minted `turnId`** (per ARCH-009 "backend owns turnId"; the WS validates it exists in the session). Confirm against the B.9c-ii turn-creation flow.
