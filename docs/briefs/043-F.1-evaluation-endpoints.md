# /tdd brief — evaluation_endpoints (F.1)

## Feature
Ship the three WER-evaluation HTTP endpoints — `GET /api/evaluation/phrases`, `POST /api/evaluation/transcribe` (STT-only), `POST /api/evaluation/wer` (compute `WerResult`, persist onto the turn when a `turnId` is supplied) — as a new thin `EvaluationController` over a new `EvaluationService`, reusing the existing `EvaluationPhraseStore` (B.6), `WerCalculator` (B.6), and the `ISttProvider` blob/REST path (C.1). **Two distinct security boundaries (this is a security-touching slice → `security-reviewer` pass on both commits):** the `/wer` hypothesis length cap before `WerCalculator.Compute` (DP-matrix DoS, ARCH-019) and the `/transcribe` audio-upload size+content-type validation (invariant #5).

## Use case + traceability
- **Task ID:** F.1
- **Architecture sections it implements:** `ARCH-009` (Evaluation APIs — the three endpoints + their request/response shapes), `ARCH-015` (WER model — normalization, algorithm, STT-only scope), `ARCH-019` (security boundaries — hypothesis DoS cap, upload validation, no `LoadError` disclosure). Touches `ARCH-016` (turn-level `WerResult` persistence tier) for the `/wer` attach.

Carry-forward: the B.6-origin hypothesis-cap MEDIUM + the `LoadError`-never-surfaced low (both already Phase-F task checkboxes).

## Acceptance criteria (what "done" means)
- [ ] `GET /api/evaluation/phrases` → `200` list of `EvaluationPhrase` (the loaded scripted phrases). When the store failed to load: `200` with an **empty list** (degrade-don't-crash) — never surfaces `LoadError`/path fragments.
- [ ] `POST /api/evaluation/transcribe` (`multipart/form-data`: `sessionId`, `phraseId`, `language`, `audio`, optional client timestamps) runs **STT only** (no translation, no TTS) and returns `{ hypothesis, sttProvider, sttModel, latencyEvents }`.
- [ ] `/transcribe` **validates the audio upload BEFORE any provider call** — size cap + content-type allowlist (reuse `CascadeUploadValidation.Validate`): oversized → `413`, empty/unsupported type → `415`, both as a sanitized `UiError` (invariant #5).
- [ ] `POST /api/evaluation/wer` (`{ sessionId, turnId?, phraseId, hypothesis }`) → `200` full `WerResult` (incl. `normalizedReference`/`normalizedHypothesis`/S/I/D/N/wer).
- [ ] **`/wer` caps the hypothesis length BEFORE `WerCalculator.Compute`** — `> ~2000 chars` (≈500 words) → `400 evaluation.invalid_phrase`, sanitized `UiError`, no matrix allocated (ARCH-019 DoS guard).
- [ ] `/wer` looks the reference up via `EvaluationPhraseStore.GetById(phraseId)`; unknown `phraseId` → `404 evaluation.phrase_not_found` (does NOT call `Compute`).
- [ ] **`/wer` persists the `WerResult` onto the turn when `turnId` is supplied** (server-authoritative) — attaches via the store, best-effort persists via `SessionPersistenceWriter`; unknown session/turn (with `turnId` given) → `404`; persist failure **degrades** (still returns the computed `WerResult` + an optional `persistenceWarning`), never crashes. When `turnId` is absent → compute-and-return only, no store write. _(See Step-2.5 Q1 — load-bearing.)_
- [ ] No `LoadError`/`Result.Error`/path/`ex.Message` fragment ever reaches any response body (sanitized `UiError` or fixed-safe message only).
- [ ] DI: `EvaluationService` + `EvaluationPhraseStore` registered; controller routed; the phrase-file path sourced from config (mirror how B.6/pricing wires its path).
- [ ] All unit tests in `server/AiInterpreter.Tests/...Evaluation...` pass; the endpoint wire (200/400/404/413/415) verified with `WebApplicationFactory`.
- [ ] `/preflight` clean (`dotnet format --verify-no-changes && dotnet build && dotnet test`).
- [ ] Cross-doc invariant updated atomic with the round: the new Transcribe/Wer DTOs → `ARCHITECTURE.md` Appendix A + the `server/CLAUDE.md` cross-doc row (orchestrator-written at `/orchestrate-end`).

## RED test outline (Step 2)

### Feature A — `GET /phrases` + `POST /wer` (commit 1; safety pin = hypothesis cap)
Tests in `EvaluationServiceTests.cs` / endpoint-wire test:

1. **`get_phrases_returns_loaded_phrases`** — Asserts: returns the store's `Phrases` verbatim. Why: ARCH-009 phrases contract.
2. **`get_phrases_store_not_loaded_returns_empty_no_error_leak`** — Asserts: a store with `IsLoaded==false` → empty list; the response body contains NO `LoadError`/path/`ex.Message` substring (sentinel-scan). Why: ARCH-019 + lesson §10 ("never surface `LoadError`").
3. **`wer_perfect_match_returns_zero`** — Asserts: `Compute` is invoked, `WerResult.Wer == 0` for hypothesis == reference. Why: ARCH-015 happy path (service delegates correctly to `WerCalculator`).
4. **`wer_unknown_phrase_id_404_no_compute`** — Asserts: unknown `phraseId` → `404 evaluation.phrase_not_found`; `WerCalculator.Compute` NOT called. Why: reference must exist before compute (no `ArgumentException` path reachable).
5. **`wer_hypothesis_over_cap_400_before_compute`** — Asserts: a hypothesis `> 2000` chars → `400 evaluation.invalid_phrase`; `WerCalculator.Compute` NOT invoked (no DP matrix allocated — assert via a spy/fake calculator or a call-count). Why: **ARCH-019 DoS cap — the security pin.** This is the test that proves the guard runs *before* the allocation.
6. **`wer_at_cap_boundary_computes`** — Asserts: a hypothesis exactly at the cap → computes (off-by-one boundary). Why: cap is inclusive/exclusive pinned deterministically.
7. **`wer_with_turn_id_attaches_and_persists`** *(if Q1 default holds)* — Asserts: `turnId` present + turn exists → the turn's `WerResult` is set; best-effort persist invoked; response carries the `WerResult`. Why: ARCH-016 turn-level WER persistence + F.3 aggregation dependency.
8. **`wer_with_unknown_turn_id_404`** *(Q1)* — Asserts: `turnId` given but session/turn unknown → `404`. Why: don't silently drop a persist target.
9. **`wer_persist_failure_degrades`** *(Q1)* — Asserts: a throwing/failing writer → still returns the `WerResult` (+ optional `persistenceWarning`); no 500. Why: degrade-don't-crash (lesson §11/§26).
10. **`wer_without_turn_id_computes_no_store_write`** *(Q1)* — Asserts: `turnId` absent → returns `WerResult`, store untouched. Why: standalone compute path.

### Feature B — `POST /transcribe` STT-only (commit 2; safety pin = upload validation)
Tests in `EvaluationServiceTests.cs` / endpoint-wire test (use a **fake `ISttProvider`** — never a live key):

11. **`transcribe_returns_hypothesis_from_stt_final`** — Asserts: a fake STT yielding `SttFinal("hello world")` → `hypothesis == "hello world"` + `sttProvider`/`sttModel` populated + `latencyEvents` non-empty. Why: ARCH-009 transcribe contract; STT-only (no translation/TTS provider touched — assert no `ITranslationProvider`/`ITtsProvider` call).
12. **`transcribe_oversized_413_no_provider_call`** — Asserts: audio length > cap → `413`; `ISttProvider.TranscribeAsync` NOT called. Why: **invariant #5 — the upload security pin** (validate before provider).
13. **`transcribe_unsupported_or_empty_type_415`** — Asserts: empty (`415`) + off-allowlist content-type (`415`); MIME params stripped (`audio/webm; codecs=opus` is accepted). Why: invariant #5 + lesson §23 (strip MIME params).
14. **`transcribe_stt_failed_maps_sanitized_error`** — Asserts: a fake STT yielding `SttFailed(providerError)` → sanitized `UiError` with the preserved code; NO `ex.Message`/payload leak. Why: ARCH-018/019 (lesson §13 — project the already-safe error, don't duplicate the table).

## Things to flag at Step 2.5
1. **(LOAD-BEARING) Does `/wer` persist server-side, or compute-and-return only?** Options: (a) compute + attach-to-turn + best-effort persist when `turnId` present, compute-only when absent; (b) compute-and-return always, persistence handled elsewhere (F.2 frontend round-trip — but the frontend cannot write session JSON, only the backend persists). My default vote: **(a)**. Rationale: `WerResult?` is turn-level, the request carries `turnId?`, ARCH-016 lists "WER results" as persisted, and F.3's comparison summary aggregates `WerSummary` from persisted turns — server-authoritative persist is the only path that reaches the session JSON. (a) adds a small `SessionStore` method + the degrade-on-persist-fail tests. **This is the one decision that changes F.1's file list (touches `SessionStore`) — confirm before GREEN.**
2. **Phrase-not-found vs hypothesis-cap error codes.** Default: hypothesis-over-cap → `400 evaluation.invalid_phrase` (per MVP_TASKS F.1); unknown `phraseId` → `404 evaluation.phrase_not_found`. My default vote: **two distinct codes as above** — a too-long hypothesis is a malformed request (400); a missing phrase is a not-found (404). Both are new `evaluation.*` codes routed through the existing `ProviderError`→`ErrorSanitizer` machinery (no bespoke sanitizer surface — mirror lesson §24's "no bespoke codes").
3. **Transcribe latency events — which to stamp, and clock source.** Default: stamp `stt.final` (always) and `stt.first_partial` (only if a partial arrives), on real arrival, via the existing factory; `clockSource` = server (the backend runs the REST STT). My default vote: **`stt.final` + optional `stt.first_partial`, server clock, never synthesized.**
4. **Upload-byte cap config key.** Default: reuse `CASCADE_MAX_UPLOAD_BYTES` (same `CascadeUploadValidation.DefaultMaxBytes` ~10MB) vs a new `EVAL_MAX_UPLOAD_BYTES`. My default vote: **reuse `CascadeUploadValidation.Validate` + its default**, but read an optional `EVAL_MAX_UPLOAD_BYTES` override (falls back to the cascade default) so eval can be tuned independently without coupling the two routes' limits. Low-stakes — take the simpler "reuse the cascade key" if the override adds noise.
5. **DTO placement.** Default: new `Evaluation/EvaluationModels.cs` for the Transcribe/Wer DTOs (keeps `SessionModels.cs` focused on domain records) vs co-locating next to `EvaluationPhrase`/`WerResult` in `SessionModels.cs`. My default vote: **a dedicated `EvaluationModels.cs`** (request/response DTOs are a different layer than the persisted domain records — mirrors how cascade keeps `CascadeModels` separate). Reuse `JsonDefaults` automatically via the MVC `AddJsonOptions` path (lesson §15).
6. **`sttProvider`/`sttModel` identity source.** Default: from the DI-resolved STT provider's config (`DeepgramOptions.Model` + the `"deepgram"` provider key) — evaluation is provider-level, not session-level. My default vote: **source from `DeepgramOptions`** (the configured STT model), not the session's `ProviderProfile`. In tests the fake provides a stand-in identity. _(Also: `language`→`SttRequest.SttLanguage` — default the phrase language string; `SourceLanguage` = the `language` enum.)_
