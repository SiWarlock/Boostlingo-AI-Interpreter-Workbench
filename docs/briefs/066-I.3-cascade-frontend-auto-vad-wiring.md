# /tdd brief — cascade_auto_vad_wiring (I.3; Phase-I cascade frontend — wire the toggle to cascade + capture-lifecycle)

> **Frontend slice (`web/`).** The cascade analogue of 063-s1: wire the existing `turnControlMode: 'manual' | 'auto'`
> toggle (063-s1, already in `sessionStore`/`SessionSetup`, transport-agnostic) to the CASCADE transport. The backend
> (062) added a per-turn WS-`start` `autoVad` flag → the cascade orchestrator auto-finalizes the turn on Deepgram's
> utterance-end (no client `stop`). This slice: send `autoVad` in the cascade WS start when auto, and handle the
> capture lifecycle so the mic stops when the turn auto-finalizes (no frontend Stop fires). Completes Phase-I cascade
> single-utterance auto-VAD (symmetric with realtime slice-1). No safety invariant.

## Feature
In auto mode, the cascade `start` frame carries `autoVad: true` (so the backend auto-finalizes on utterance-end), and
the frontend STOPS THE AUDIO CAPTURE when the backend auto-finalizes (the `done` frame), since no manual Stop fires.
The Stop button becomes an early-end. Manual mode unchanged.

## Design (mostly wiring + the capture-lifecycle fix)
1. **Send `autoVad` in the start frame** — `startRecording` reads `store.getState().turnControlMode`; passes
   `autoVad: turnControlMode === 'auto'` to `client.start`. `buildStartFrame` includes it.
2. **⭐ Stop the capture on the auto-finalize (auto mode).** When the backend auto-finalizes (the `done` frame in auto
   mode), the capture must stop (no frontend Stop fires; the WS is closing). Wire `recordingActions` to stop
   `captureHandle` when the turn auto-completes — e.g. the `cascadeStreamClient` calls an injected `onTerminal`/`onDone`
   callback (or `recordingActions` reacts to the terminal) that stops the capture + clears the handle, idempotent with a
   manual `stopRecording`. (Confirm the cleanest wiring at Step-2.5 — Q1.)
3. **Stop button in auto mode = early-end.** A manual Stop still works (force-finalize before utterance-end): send
   `client.stop()` (the backend idempotent-finalizes — whichever of {utterance-end, stop} fires first wins, 062/§22) +
   stop the capture. So Stop is OPTIONAL in auto (the turn auto-ends on silence) but functional as an early-end. (Confirm
   vs disabling Stop in auto — Q2.)
4. **Manual mode unchanged** — `autoVad` false (omitted); Stop sends `client.stop()` + stops capture exactly as today.
