# /tdd brief — cascade_ws_hardening (C.4b)

## Feature
Harden the cascade streaming WebSocket boundary that landed in C.4a (`WS /api/cascade/stream` → `CascadeWebSocketEndpoint` → the unchanged B.4 orchestrator). C.4a shipped the core integration clean and deliberately deferred eight boundary-robustness items; this slice lands them: **(1) bound the audio `Channel<AudioFrame>`** [SECURITY], **(2) validate the WS `Origin`** [SECURITY], **(3) the robustness bundle** — `TtsFirstAudio.ContentType` clamp, double-`End`/`complete` idempotency guard, stream-without-terminal `<stage>.unknown` hardening, `start`-frame field validation + error-shape polish, `VoiceByLanguage` resolution + stale-comment fix, and unit-pinning the multi-segment cost accumulation.

## Acceptance criteria (what "done" means)

**Commit 1 — bounded audio channel (SECURITY):**
- [ ] The C.4a unbounded `Channel.CreateUnbounded<AudioFrame>()` (`CascadeWebSocketEndpoint.cs:88`) becomes a **bounded** channel with a chosen capacity + full-mode (Q1); a stalled STT consumer can no longer grow the buffer without bound (the G.4 5-min-no-leak tie). The receive pump's write strategy is updated accordingly (`TryWrite` → the chosen strategy).
- [ ] **No audio is silently lost in the happy path** (the chosen full-mode preserves transcription fidelity when the consumer keeps up — Q1).

**Commit 2 — Origin validation (SECURITY):**
- [ ] `CascadeWebSocketEndpoint.HandleAsync` validates the `Origin` request header against the allowed origin (`FRONTEND_ORIGIN`, the same value CORS uses) **before** `AcceptWebSocketAsync`; a disallowed/missing Origin is rejected with `403` and **no socket is accepted**.
- [ ] The allow/deny decision is a **pure, unit-tested** helper (`IsAllowedOrigin(origin, allowed)`): exact match allowed; null/empty/whitespace Origin rejected; case + trailing-slash behavior pinned (Q2).

**Commit 3 — robustness bundle:**
- [ ] **ContentType clamp:** the provider-sourced `Audio.ContentType` is clamped to a known audio allowlist (Q6) **before** it is serialized into the WS `audio` message; an unknown/garbage content-type falls back to the default (`audio/mpeg`). Pure + unit-tested in `CascadeWsMapping`.
- [ ] **Stream-without-terminal hardening:** in `CascadeStreamingOrchestrator`, a **started** translation or TTS sub-stage whose stream **ends without its terminal event** (`TranslationFinal` / `TtsComplete`) now yields a `<stage>.unknown` `Error` + `Done(Failed)` instead of silently skipping downstream / completing (current behavior: `TranslateSegmentAsync:247` / `SynthesizeSegmentAsync:331` `yield break` with no error). STT-stream-without-final behavior settled at Q4. Added via a new `ProviderErrorMapper.Unknown(provider, stage)` non-exception factory (fixed `SafeMessage`; never echoes provider text — lesson §5/§13).
- [ ] **Double-`End`/`complete` guard:** completing/ending an already-terminal turn (or ending an already-ended session) is **idempotent** — the second call does not re-overwrite a terminal status or double-persist (Q3). Now relevant because the WS terminal path can collide with the HTTP `/end`·`/complete` paths. Unit-tested at the chosen layer.
- [ ] **Start-frame field validation:** `CascadeStartValidation.ParseStart` rejects a missing/blank `sessionId` or `turnId` and a `sampleRate <= 0` (today they fall through to empty-string/0); the malformed-JSON vs encoding-reject vs field-reject messages are distinguished via a single parametrized `Invalid(...)` factory (error code per Q5). The oversized (>16KB) / fragmented start frame is rejected at the shell receive (manual-smoke; `!EndOfMessage` guard). Non-WS request → a `UiError`-shaped `400` body (today `HandleAsync` returns a bare 400). Pure validation unit-tested.
- [ ] **`VoiceByLanguage` resolution:** the `OpenAiTtsOptions.VoiceByLanguage` map resolves to the effective TTS voice via a pure `ResolveVoice(requestVoice, targetLanguage, options)` helper in `OpenAiTtsMapping`, applied in `OpenAiTtsProvider` (precedence per Q7); the stale `// unused until ... (C.3)` comment in `Providers/OpenAI/OpenAiOptions.cs:36` is corrected.
- [ ] **Multi-segment cost accumulation unit-pin:** the inline per-segment token/char accumulation (`CascadeWebSocketEndpoint.cs:100-118`) is extracted to a pure fold helper in `CascadeWsMapping` and **unit-tested for multi-segment summing** (pins the C.4a additive-accumulation undercount fix, currently shell-only/manual-smoke).

**All commits:**
- [ ] All new/extended unit tests pass; full suite green (211 + new); `dotnet format` clean; `dotnet build` 0W/0E; `/preflight` clean.
- [ ] **No leak:** Origin/field-validation rejections, the clamp, and the `Unknown` factory introduce no `ex.Message`/key/stack/raw-payload into any WS message or persisted JSON (invariant #4; the existing sentinel + sanitizer hold).
- [ ] Cross-doc: **none expected** for typed-model/Appendix-A contracts. **If** Q5 introduces a new normalized error code (`cascade.invalid_request`), that is an **ARCH-018 architecture-doc note** (not a cross-doc invariant/model change) — flag at Step 9.

## Things to flag at Step 2.5
1. **Bounded-channel capacity + full-mode.** Backpressure (`await WriteAsync` — the pump blocks the socket read when full, capping memory; loses no audio) vs `DropOldest`/`DropWrite` (never blocks, but drops audio → degrades transcription). My default vote: **bounded + backpressure (`await writer.WriteAsync(ct)`)**, capacity ≈ a few seconds of 20–50ms frames (e.g. **256**, ~5–12s) — dropping audio corrupts the cascade output; backpressure is correct for a single-user MVP. **And:** treat the capacity/full-mode as a **manual-smoke shell decision** (like the C.4a channel), NOT a forced unit test — extract a tiny write-strategy helper + test ONLY if the team wants the pin. Confirm capacity + mode + whether to extract.
2. **Origin helper placement + reject shape.** Pure `IsAllowedOrigin` in its **own `CascadeOriginValidation.cs`** vs folded into `CascadeStartValidation`. My default vote: **own file** (Origin is an auth boundary distinct from start-frame parsing; mirrors the C.4a "validation in its own file for commit isolation" precedent). Reject shape: **`403` status, no socket accept**; body = empty 403 vs a `UiError`-shaped body. Default: **403 + a `UiError`-shaped body** for consistency with the non-WS-400 polish. Confirm. (Also: trailing-slash + case — default to **exact ordinal match** against `FRONTEND_ORIGIN`, no normalization, since browsers send a canonical Origin.)
3. **Double-end/complete guard layer + contract.** At the **`SessionStore`** (a status-aware guard: completing an already-terminal turn is a no-op returning the existing turn) vs at the **WS endpoint** (skip the terminal persist if the turn is already terminal). My default vote: **store-level idempotency** (one guard covers both the WS and HTTP collision paths; the endpoint stays thin) — completing an already-`Completed`/`Failed` turn returns the existing turn unchanged + does not re-persist. Confirm layer + whether a re-complete returns existing (default) or signals a conflict.
4. **STT stream-without-final semantics.** A real STT stream that ends with **zero** `SttFinal` (e.g. silence, or a lost final): treat the turn as **`Completed` (empty, no error)** vs **`stt.unknown` `Failed`**. My default vote: **harden only the started-stage-without-terminal cases for translation/TTS** (the clear bug — a stage that began but never finished); for STT, **a clean end with no final stays `Completed`** (recording silence is valid) UNLESS partials were seen but no final arrived, in which case → **`stt.unknown` Failed** (a lost final is a real failure). Confirm the STT rule; it's the subtlest of the three.
5. **Error code for start-frame field rejections.** Reuse **`cascade.invalid_audio`** (no new code, cross-doc stays fully "none") vs a new **`cascade.invalid_request`** (semantically honest — a missing `sessionId` isn't "invalid audio"). My default vote: **`cascade.invalid_request`** (honesty; it's an ARCH-018 doc note, not a typed-contract change — cheap) — but if you prefer zero doc surface on this bundle, reuse `cascade.invalid_audio` and note the imprecision. Confirm.
6. **ContentType clamp allowlist.** The allowed audio content-types to pass through (others → `audio/mpeg`). My default vote: **`{audio/mpeg, audio/wav, audio/L16, audio/pcm, audio/ogg}`** (covers the C.3 `ResponseFormat` map mp3/wav/pcm + headroom), case-insensitive compare, default fallback `audio/mpeg`. Confirm the set.
7. **`ResolveVoice` precedence + placement.** Placement: **in `OpenAiTtsMapping`/`OpenAiTtsProvider`** (the provider owns `OpenAiTtsOptions` + the request's target language + requested voice — keeps the orchestrator provider-agnostic) vs upstream at the WS boundary. My default vote: **in the TTS provider** (correct layering — the orchestrator must not learn about `VoiceByLanguage`). Precedence: **explicit non-empty `request.Voice` → `VoiceByLanguage[targetLang]` → `options.Voice` default.** Confirm precedence + placement.
8. **Robustness-bundle commit count.** Default: **1 robustness commit** (Commit 3) per the lead's "robustness bundled" directive. **You MAY split it into ≤3 commits at natural seams if Step-2.5 review finds the single bundle too large to grok in one sitting** — suggested seams: (3a) orchestrator stream-without-terminal + `Unknown` factory; (3b) WS-boundary items (ContentType clamp + start-frame validation + double-end guard + cost fold); (3c) TTS `ResolveVoice` + stale comment. Your call at Step 2.5; do NOT spawn a separate brief. The two security items (C1 bounded channel, C2 Origin) stay **isolated own-commits — non-negotiable**.
