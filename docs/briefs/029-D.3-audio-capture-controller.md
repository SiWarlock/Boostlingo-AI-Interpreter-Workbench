# /tdd brief — audio_capture_controller

## Feature
The audio capture controller: a streaming path (`getUserMedia` → `AudioContext` → `AudioWorkletNode` → Float32→**linear16** Int16 frames, ~20–50ms, declared at the actual `AudioContext.sampleRate`, **no resample**) and a blob fallback (`MediaRecorder` with a **probed** mimeType → one blob), plus the mic-permission-denied path. Most of this is browser-API shell (manual-smoke per ARCH-020); the deterministic cores — the Float32→Int16 conversion, the MediaRecorder mimeType probe-order, and the mic-error→`UiError` mapping — are **test-first**.

## Use case + traceability
- **Task ID:** D.3
- **Architecture sections it implements:** `ARCH-030` (§4 Audio Capture & Format — the single home for capture API + format + the no-transcode decision), `ARCH-007` (the capture controller is one of the frontend's owned responsibilities; clean separation).
- **Related context:** Builds on D.1/D.2 (store, `UiError` type). **The controller is consumed in D.4** (the cascade streaming client sends the frames over the WS; `RecordingControls` drives start/stop) — so D.3 is again foundation-ahead-of-callers (honest, closed at D.4 + the Phase-D-exit audit). The two formal component tests (incl. **mic-denied → error status + Start disabled**) land in **D.7**.

## TDD scope (read first — this slice is mostly manual-smoke)
**Test-first (deterministic, pure):**
- `floatTo16BitPCM(Float32Array) → Int16Array` — clamp to [-1, 1], scale to Int16 (the linear16 conversion the worklet does).
- `probeRecorderMimeType(isTypeSupported) → string | null` — the ARCH-030 probe order `audio/webm;codecs=opus` → `audio/webm` → `audio/mp4` → `audio/ogg;codecs=opus`; `null` if none supported.
- `micErrorToUiError(err) → UiError` — `NotAllowedError`/`SecurityError` → `mic.permission_denied`; `NotFoundError` → `mic.not_found`; else a generic `mic.unavailable`; never leaks the raw error (lesson §3 posture).

**Exempt (manual-smoke / ARCH-020 — browser internals):** `getUserMedia`, `AudioContext`, `AudioWorkletNode` wiring, the `pcmWorklet` processor running in the worklet realm, the `MediaRecorder` lifecycle, the actual frame emission. These are the capture *shell* — TDD their pure helpers (above), smoke the wiring in the browser (the C/D demo-checklist).

## RED test outline (Step 2)

### pcm (`web/src/audio/pcm.test.ts`)
1. **`float_to_int16_scales_and_clamps`** — Asserts: `0 → 0`; `1.0 → 32767`; `-1.0 → -32768`; out-of-range (`1.5`/`-1.5`) clamps to the int16 bounds (no wraparound). Why: linear16 correctness (ARCH-030) — a wrap would corrupt audio sent to Deepgram.
2. **`float_to_int16_length_preserved`** — Asserts: output length === input length; an empty input → empty output. Why: frame-size integrity.

### captureSupport (`web/src/audio/captureSupport.test.ts`)
3. **`probe_returns_first_supported_in_order`** — Asserts: with a fake `isTypeSupported` that supports only `audio/mp4`, returns `audio/mp4`; when webm;opus is supported, returns it first (order honored). Why: ARCH-030 probe order (Safari<18.4 = mp4, not webm).
4. **`probe_returns_null_when_none_supported`** — Asserts: nothing supported → `null` (caller surfaces an error). Why: total function.
5. **`mic_error_maps_to_uierror_by_name`** — Asserts: `{name:'NotAllowedError'}` → `mic.permission_denied`; `{name:'NotFoundError'}` → `mic.not_found`; an unknown error → `mic.unavailable`; in all cases `safeMessage` is a fixed string and the raw error message is NOT echoed. Why: actionable + no-leak (ARCH-018; lesson §3).

> `audioCaptureController` + `pcmWorklet` are exempt (manual-smoke). If a deterministic seam emerges that's worth pinning (e.g. a pure frame-accumulator), flag at Step 2.5.

## Things to flag at Step 2.5
1. **Extract the pure helpers (`pcm.ts`, `captureSupport.ts`) from the worklet/controller?** Default vote: **yes** — the worklet runs in a separate JS realm (can't be unit-tested directly), so the conversion must live in an importable pure module; the probe + mic-mapping are likewise pure. This is the testable seam.
2. **Controller callback contract for D.4.** Default vote: a factory `createAudioCaptureController()` exposing `startStreaming({ onFrame, onError }) → { sampleRate, encoding:'linear16', stop() }` + `recordBlob() → Promise<{ blob, mimeType }>`. `onFrame` hands a binary frame (Int16 `ArrayBuffer`) so D.4 sends it straight over the WS. Injectable `getUserMedia`/`AudioContext`/`MediaRecorder` where it helps a smoke harness, but the controller stays manual-smoke. (Refine when D.4 consumes it.)
3. **linear16 frame size.** Default vote: target **~20ms** frames (accumulate the worklet's 128-sample blocks to ~20ms at the actual rate); within the ARCH-030 20–50ms band; smaller = lower latency. A named const.
4. **Mic-denied `UiError` code.** Default vote: **`mic.permission_denied`** (+ `mic.not_found`/`mic.unavailable`) — frontend-coined (getUserMedia never reaches the backend), aligned with the ARCH-018 `MicPermissionDenied` overlay name.
5. **Worklet module loading under Vite.** Default vote: `audioContext.audioWorklet.addModule(new URL('./pcmWorklet.ts', import.meta.url))` (Vite-native). Flag if a `vite.config.ts` tweak is needed.
