# /tdd brief — eval_turn_exclusion (F.4)

## Feature
Mark standalone WER-evaluation turns with an explicit `IsEvaluation` flag (set server-side at `POST /api/evaluation/wer`) and **exclude them from `ModeSummary`** (per-mode `TurnCount` + the per-mode averages + error count) in `SessionSummaryService`, so F.3's comparison turn counts are **exact** — while keeping them in `WerSummary` (the WER comes from eval turns). **F.2's frontend is byte-identical** (it already calls `/wer` with the turnId; the marker is set inside the existing attach transform).

F.3 reads `ModeSummary.TurnCount` live from `GET /summary` → **no F.3 change**; once the backend excludes eval turns, F.3 auto-shows exact counts. ## Things to flag at Step 2.5
1. **Top-level `SessionSummary.TurnCount` — exclude eval turns or count all?** `SummarizeMode` (per-mode) MUST exclude (the comparison fix). The top-level `SessionSummary.TurnCount = session.Turns.Count` is a separate "total turns" number. My default vote: **count all at the top level** (it's an honest "total turns in the session incl. evaluations") + exclude only per-mode (where the comparison reads). If you'd rather the top-level also exclude (a "total interpretation turns" semantic), flag — but per-mode is where F.3's comparison counts come from, so per-mode is the load-bearing one. Either way, document the chosen semantic.
2. **`ErrorCount` — exclude eval turns' errors from `ModeSummary.ErrorCount`?** My default vote: **yes** (consistent — an eval turn isn't an interpretation turn, so its errors don't belong in the mode's interpretation error count). The `.Where(… && !t.IsEvaluation)` filter naturally covers it (one filter, all per-mode fields).
3. **Set the marker at `/wer` (default) vs at create.** Setting at `/wer` keeps **F.2 unchanged** (the user's constraint) but leaves the orphaned-eval-turn (computeWer-failed → unmarked) counted — the bounded carry-forward residual. Setting at create would need an F.2 signal (a create-time flag) → **rejected** (violates "F.2 frontend identical"). My default vote: **set at `/wer`** + leave the orphan as the documented residual.
