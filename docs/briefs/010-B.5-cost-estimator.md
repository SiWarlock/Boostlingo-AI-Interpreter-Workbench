# /tdd brief — cost_estimator (B.5)

## Feature
`CostEstimator` — loads the (already-bound) pricing config, estimates per-turn cost by **branching on pricing basis** (audio-minute vs tokens vs audio-output-tokens vs characters), converts realtime audio-seconds→tokens before applying token rates, computes cost/min, and emits a `CostEstimate` (per-stage for realtime, aggregated for a cascade turn) with `assumptions` + `pricingConfigVersion`. Missing/unavailable pricing degrades to "estimate unavailable" — never a crash.

## Acceptance criteria (what "done" means)
- [ ] **Audio-minute basis (Deepgram STT):** given audio minutes → `EstimatedUsd = minutes × effectiveUsdPerAudioMinute`; `PricingBasis = "usd_per_audio_minute"`.
- [ ] **Tokens basis (OpenAI translation):** given input/output tokens → `EstimatedUsd = inputTokens/1e6 × inputRate + outputTokens/1e6 × outputRate`; `PricingBasis = "tokens"`; model selectable (`gpt-5.4-nano`/`-mini`), recorded in `Model`.
- [ ] **Audio-output-tokens basis (gpt-4o-mini-tts):** given audio-output tokens (and optional text-input tokens) → cost from `audioOutputUsdPerMillionTokens` (+ `textInputUsdPerMillionTokens`); `PricingBasis = "audio_output_tokens"`. Fallback to `approxUsdPerAudioMinute × minutes` when token counts aren't available (flag at 2.5).
- [ ] **Characters basis (tts-1/tts-1-hd):** given character count → `EstimatedUsd = chars/1e6 × usdPerMillionCharacters`; `PricingBasis = "characters"`.
- [ ] **Realtime:** converts audio-seconds → tokens via a documented conversion factor (estimate — see 2.5 Q3) before applying `audioInput/audioOutput UsdPerMillionTokens`; `Model` records the realtime model; cached-input rate applied when present.
- [ ] **Per-turn aggregation:** a cascade turn yields ONE aggregate `CostEstimate` (sum of STT+translation+TTS) with the per-stage breakdown in `Units`; a realtime turn yields one `CostEstimate`. (Aggregation shape → 2.5 Q1.)
- [ ] **cost/min:** `EstimatedUsdPerMinute = EstimatedUsd ÷ (audioDurationMs / 60000)` when `audioDurationMs > 0`, else `null`.
- [ ] **Degrade:** missing/unavailable pricing (loader `Result.Failure`, or the model/basis absent from config) → "estimate unavailable" (no crash, no partial-garbage estimate). Shape → 2.5 Q6.
- [ ] Every `CostEstimate` carries `PricingConfigVersion` + an `Assumptions` entry ("Estimate from configured public pricing, not provider invoice data.").
- [ ] All unit tests in `AiInterpreter.Tests/CostEstimatorTests.cs` pass.
- [ ] `/preflight` clean.

## Things to flag at Step 2.5
1. **Cascade per-turn aggregation shape.** The turn carries ONE `CostEstimate`; cascade has 3 stages. My default vote: **one composite `CostEstimate`** — `Provider = "cascade"`, `Model = <translation model used>` (the cascade comparison axis per ARCH-014 — lets B.7 group cascade cost by translation-model variant), `PricingBasis = "composite"`, `EstimatedUsd = sttUsd + translationUsd + ttsUsd`, `Units` = `{ "sttUsd":…, "translationUsd":…, "ttsUsd":…, plus the raw unit counts }`, `Assumptions` lists each stage's basis. (Realtime turn = a single non-composite estimate.) Push back if you'd rather store per-stage estimates — but the turn model + WS message are singular, so something must aggregate, and the estimator is the right owner (cost logic stays out of the WS handler).
2. **Usage input shape.** My default vote: a single **`CostUsage` record with nullable fields** (`AudioMinutes?`, `InputTokens?`, `OutputTokens?`, `Characters?`, `AudioOutputTokens?`, `TextInputTokens?`, `AudioInputSeconds?`, `AudioOutputSeconds?`); the basis determines which are read. Cleaner than per-stage method overloads and extends to realtime.
3. **Realtime audio-seconds→tokens conversion factor.** `pricing.json` has no explicit factor (the `estimatorNote` says "factors are estimates — CONFIRM at build"). My default vote: **a documented constant in the estimator** (e.g. the OpenAI realtime audio token rate ≈ tokens/sec), clearly commented as an estimate, and added to the build-time-confirm list (Decisions tabled already lists "realtime token-conversion factors"). Alternative: add a `tokensPerAudioSecond` field to `pricing.json` realtime block — heavier (touches A.4's bound shape). Default = constant for now; promote to config if E needs it tunable.
4. **gpt-4o-mini-tts when audio-output-token counts aren't available.** TTS audio-output tokens may not be known at estimate time (the provider streams audio, not a token count). My default vote: **prefer real audio-output tokens when present; else fall back to `approxUsdPerAudioMinute × minutes`** (the config carries `approxUsdPerAudioMinute` for exactly this), recording the fallback in `Assumptions`.
5. **Degrade representation.** My default vote: **return `Result<CostEstimate>`** (mirrors A.4 `PricingLoader`, lesson §3); the C.4 caller emits no `cost` message (or an "unavailable" UI state) on `Failure`. Alternative: a `CostEstimate` with a sentinel/empty — rejected (a fake-zero estimate is worse than an honest "unavailable").
6. **Rounding / decimal.** My default vote: **compute in `decimal`, do NOT round** in the estimator (the UI formats for display). Keeps tests exact and avoids premature precision loss.
