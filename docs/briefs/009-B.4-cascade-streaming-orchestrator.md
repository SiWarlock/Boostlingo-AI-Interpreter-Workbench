# /tdd brief — cascade_streaming_orchestrator (B.4)

## Feature
`CascadeStreamingOrchestrator` — drives the streaming cascade pipeline `STT (partials/finals) → per-finalized-segment translation (streamed) → TTS (streamed)` exactly per ARCH-011, stamping `LatencyEvent`s on the **real first arrival** of each event type (via B.3's `LatencyEventFactory`), emitting normalized `TranscriptSegment`s + audio chunks + latency + errors to an output stream, and enforcing the empty-transcript short-circuit, the two partial-failure rules, and per-stage timeout/cancellation. Tested end-to-end against the B.2 fakes — no real keys.

## Streaming-honesty rule (LOAD-BEARING — read before writing code)
The cascade output path must **genuinely stream**: the orchestrator `await foreach`-consumes each provider's `IAsyncEnumerable`, emits source partials as they arrive, target tokens as they arrive, and TTS audio chunks as they arrive — all **before** the terminal `Done`. Never buffer a whole stage and relabel a completion as `first_*`; never collapse the translation stage to a blocking call (ARCH-011/013; area forbidden-patterns #3/#4). This is the project's load-bearing honesty invariant — the success-path test pins it directly (target text + first audio emitted before `Done`). _(Not one of the 5 root safety rules, so no mandatory `security-reviewer` gate — but treat it as non-negotiable.)_

## Acceptance criteria (what "done" means)
- [ ] `CascadeStreamingOrchestrator.RunAsync(...)` returns an `IAsyncEnumerable<CascadeOutputEvent>` that, for a multi-word success turn, emits in order: source `Transcript`(partial→final) → target `Transcript`(partial→final) → `Audio`(first chunk … chunks) → `Done(Completed)`, with `Latency` events interleaved on first arrival.
- [ ] **Streaming contract:** target `Transcript` and the first `Audio` are emitted **before** `Done` (proxy for "before `tts.complete`"); source partials precede the source final.
- [ ] Stamps (via `LatencyEventFactory.Stamp`, `clockSource: server`): `cascade.audio.received`, `stt.started`, `stt.first_partial` (first partial only), `stt.final`, `translation.started`, `translation.first_token` (first delta only), `translation.final`, `tts.started`, `tts.first_audio`, `tts.complete`, `turn.completed` — each on real first arrival, name from `LatencyEventNames`.
- [ ] **Empty-transcript short-circuit:** an empty/whitespace `SttFinal` → emit `Error(cascade.empty_transcript)` + `Done(Failed)`; **translation + TTS are NEVER invoked** (proven by call-count spies == 0).
- [ ] **Partial-failure — translation fails:** source `Transcript` kept; no target; **TTS never invoked**; `Error(<translation code>)`; `Done(Failed)`.
- [ ] **Partial-failure — TTS fails:** source + target `Transcript`s kept; no `Audio`; `Error(<tts code>)`; `Done(Failed)`.
- [ ] **Per-stage timeout:** each stage wrapped in `CancellationTokenSource.CreateLinkedTokenSource(ct).CancelAfter(TimeoutSeconds)`; on `OperationCanceledException` → `Error(<stage>.timeout, Retryable=true)` (via `ProviderErrorMapper.Timeout`), downstream skipped per partial-failure rules, `Done(Failed)`.
- [ ] All unit tests in `AiInterpreter.Tests/CascadeOrchestratorTests.cs` pass.
- [ ] `/preflight` clean (`dotnet format --verify-no-changes && dotnet build` WAE 0W/0E `&& dotnet test`).

## RED test outline (Step 2) — `AiInterpreter.Tests/CascadeOrchestratorTests.cs`
CRITICAL tier (ARCH-020) — the orchestrator is the spec's centerpiece; cover thoroughly.

1. **`success_path_streams_segments_and_audio_in_order`** — fakes: STT (partials→final), translation (tokens→final), TTS (chunks→complete). Assert output order (source partial(s) → source final → target partial(s) → target final → first Audio → Audio chunks → Done(Completed)); **target Transcript + first Audio precede Done** (streaming contract); `isFinal` flags correct; latency events present (stt.started/final, translation.started/final, tts.started/first_audio, turn.completed) with names from `LatencyEventNames`.
   - Why: ARCH-011 pipeline + streaming contract.
2. **`empty_transcript_short_circuits_no_translation_or_tts`** — STT emits empty `SttFinal`; translation + TTS are **spy** providers. Assert `Error(cascade.empty_transcript)` + `Done(Failed)`; **translationSpy.Calls == 0 && ttsSpy.Calls == 0**.
   - Why: ARCH-011 empty-transcript rule (the call-count spies are the teeth).
3. **`translation_failure_keeps_source_skips_tts`** — translation fake = error variant; TTS = spy. Assert source `Transcript` present, no target `Transcript`, **ttsSpy.Calls == 0**, `Error(<translation code>)`, `Done(Failed)`.
   - Why: ARCH-018 partial-failure (translation).
4. **`tts_failure_keeps_both_transcripts_audio_unavailable`** — TTS fake = error variant. Assert source + target `Transcript`s present, **no `Audio`**, `Error(<tts code>)`, `Done(Failed)`.
   - Why: ARCH-018 partial-failure (TTS).
5. **`stage_timeout_emits_retryable_timeout_error_and_skips_downstream`** — a stage fake delays past a short configured `TimeoutSeconds`; linked CTS `CancelAfter` fires. Assert `Error(<stage>.timeout, Retryable==true)`, downstream stage not invoked (spy == 0), `Done(Failed)`.
   - Why: ARCH-012 timeout/cancellation.
6. **`stamps_latency_on_first_arrival_not_synthesized`** — drive a stream with multiple partials/deltas; assert exactly **one** `stt.first_partial` / `translation.first_token` / `tts.first_audio` (the first), and that their timestamps equal the first event's arrival, not a back-dated/relabeled completion.
   - Why: streaming-honesty (forbidden-pattern #3; lesson §7). (May fold into #1 if leaner.)

## Things to flag at Step 2.5
1. **Output mechanism — `IAsyncEnumerable<CascadeOutputEvent>` vs `ChannelReader<CascadeOutputEvent>`?** My default vote: **`IAsyncEnumerable<CascadeOutputEvent>`** via `yield return` — consistent with the provider interfaces, trivially testable (`await foreach` → list), and C.4 adapts it to the WS. ARCH-011 says "output channel" loosely; the async-enumerable is the idiomatic realization. (A `Channel<T>` is the alternative if concurrent produce/consume is needed — but the MVP pipeline is sequential per finalized segment, so it isn't.)
2. **`relativeMs` origin for server-stamped events.** My default vote: **`cascade.audio.received` timestamp** (captured when the first `AudioFrame`/STT start is observed) is the per-turn server origin; all server events' `relativeMs` are relative to it. (`turn.recording.started`/`.stopped` are browser events supplied by the client at C.4 — not available to the server-side orchestrator.)
3. **Asserting "translation/TTS never invoked".** My default vote: **minimal call-counting spy decorators** in the Tests project (`CountingTranslationProvider`/`CountingTtsProvider` wrapping a fake, exposing `Calls`). Cleaner than a throw-if-called fake and reusable across tests 2/3/5.
4. **Timeout-test determinism.** `CancelAfter` uses a real wall-clock timer, so the timeout test isn't fully clock-injectable. My default vote: **short real `TimeoutSeconds` (e.g. 50ms) + a fake stage delay well past it (e.g. 500ms)** with a wide margin → non-flaky; assert the OCE→`<stage>.timeout` mapping + downstream skip. (Abstracting `CancelAfter` behind a seam is over-engineering for one test.)
5. **B.4 output scope — confirm cost/persist/WS are deferred.** My default vote: **B.4 emits `Transcript`/`Latency`/`Audio`/`Error`/`Done(status)` only.** Cost (`Done` carries no cost; B.5 computes, C.4 emits), persistence (B.7), and the WS framing (C.4) are out of scope. The orchestrator is provider-and-transport-agnostic — it knows nothing about WebSockets or JSON.
6. **`Audio` carries raw `byte[]` (forward note, no B.4 action).** Streaming raw TTS bytes to the browser is the whole point and is fine; safety rule #3 (raw audio **never persisted**) bites at B.7, not here — B.4 doesn't persist. Just don't let `CascadeOutputEvent.Audio` bytes leak into any session model. Flag at Step 9 only if you see a persistence path sneaking in.
