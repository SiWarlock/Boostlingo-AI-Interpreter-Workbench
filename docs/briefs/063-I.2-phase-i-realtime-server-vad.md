## Feature
A session-level **turn-control toggle (Manual | Auto-VAD)** that KEEPS the existing manual flow and adds an auto-VAD
mode: in auto mode the realtime controller sends `turn_detection: { type: "server_vad", … }` in `session.update`,
does NOT send the manual `input_audio_buffer.commit` / `response.create` on Stop, and handles the server's auto-turn
events (speech start/end → auto-commit → auto-response) as `InterpretationTurn`s.

## Design intent (confirm/refine at Step 2.5 after the Step-1 verification)
- **Turn-control mode = store state** (`turnControlMode: 'manual' | 'auto'`, session-scoped; default `'manual'` —
  preserve current behavior). A UI toggle (KEEP the manual option, web §22 "adjust a query never an assertion").
- **Controller branches on the mode:**
  - **Manual (unchanged):** the current flow (`turn_detection: null` + commit/response.create on Stop). Regression-pin.
  - **Auto-VAD:** Start sends `session.update { turn_detection: { type:'server_vad', … }, transcription: {model} }` —
    **MUST still re-assert `transcription` (the 053-B clobber fix — else the source transcript regresses)**. Stop does
    NOT send commit/response.create (the server owns segmentation); Stop may be a no-op or an early-end (flag Q3).
  - **Auto-turn lifecycle:** the server auto-detects speech end → auto-commits → auto-creates a response. The controller
    must map each auto-detected segment to an `InterpretationTurn` (begin a turn on `speech_started`/`response.created`,
    finalize on `response.done`). **The C2b `/complete` cost POST + the `/events` report MUST still fire per auto-turn**
    (web §26 — so auto-turns get priced + their metrics reported).
- **Invariant #3 preserved:** audio still plays via the WebRTC media track (`pc.ontrack`); the sink stays store-only,
  no audio bytes (web §16). `ek_` stays a transient client local (invariant #2).
