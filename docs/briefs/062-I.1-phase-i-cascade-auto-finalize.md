# /tdd brief — cascade_auto_finalize (I.1; Phase-I backend, cascade half)

> **Backend slice (`server/`).** Phase I = automatic voice-activity detection, both modes, **keeping the manual
> toggle**. This is the **cascade half**: auto-finalize the cascade turn on Deepgram's utterance-end signal (so a turn
> ends on detected silence, NOT only on a client `stop` frame). The realtime half is FE-led + independent (063).
> **Phase I REVISES ARCH-003's locked "turn-based click start/stop, no VAD tuning" decision** — user-approved
> ("Phase I up, keep all scope"). Touches the streaming-completion path (server lessons §21/§22) → careful + invariant-
> pinned. No NEW safety invariant, but the streaming-honesty + idempotent-finalize invariants are load-bearing here.

## Feature
When auto-VAD is active for a cascade turn, the orchestrator/WS endpoint **auto-finalizes the turn on Deepgram's
utterance-end** (detected silence) instead of waiting for the client `stop` frame. The manual `stop` still works as an
**early-end**. Manual mode (current behavior — finalize on client `stop`) is preserved.

## Design intent (confirm/refine at Step 2.5 after the Step-1 pre-orient)
- **Auto-finalize on Deepgram utterance-end** when auto-VAD is active: the orchestrator treats the utterance-end signal
  (NOT the client `stop`) as the turn-terminal → routes through the **same idempotent `FinalizeTurn`** (§22 — never a
  second finalize path; every caller through the one seam). Manual `stop` remains a valid early-terminal (idempotent —
  whichever fires first wins; the other is a no-op).
- **Streaming honesty preserved (§8 / forbidden-pattern #3):** stamp `stt.final` on the REAL Deepgram final only; do NOT
  synthesize a turn-end. The auto-finalize is driven by a REAL provider signal, never a wall-clock timer the orchestrator
  invents (a timer-based "silence" the provider didn't report = a synthetic metric → a Finding).
- **Manual mode unchanged:** when auto-VAD is OFF, finalize on client `stop` exactly as today (regression-pin it).
- **Degrade/fail-closed (§22):** a terminal-less real stream still fails closed (`cascade.unknown`); a clean no-final
  stream is valid silence→Completed; the bounded audio channel + writer-complete-on-Done invariants hold.

## Acceptance (RED-first; deterministic — against scripted-finals/utterance-end fakes, never live Deepgram)
- [ ] **Auto-VAD on: an utterance-end signal finalizes the turn** through `FinalizeTurn` (no client `stop` needed) — RED
      via a scripted STT fake emitting partials→final→utterance-end → assert the turn reaches the terminal `Done`/Completed
      with its assembled transcript+metrics+cost, exactly as a `stop`-finalized turn would.
- [ ] **Idempotent terminal (§22):** an utterance-end auto-finalize THEN a late client `stop` (or vice-versa) finalizes
      **once** — the second is a no-op (`FinalizeTurn→{Turn, Applied:false}`). Pin both orders.
- [ ] **Manual mode regression:** auto-VAD OFF → finalize on client `stop` only; an utterance-end signal does NOT
      auto-finalize (the existing behavior is preserved). Pin it.
- [ ] **Streaming honesty:** `stt.final` stamped on the real final only; NO synthesized/timer-based turn-end (assert no
      fabricated stamp; the auto-finalize rides a real provider signal).
- [ ] **Fail-closed + silence (§22) still hold:** terminal-less real stream → `cascade.unknown`; clean-no-final → valid
      silence Completed. Don't regress §22's pins.
- [ ] `/preflight` (backend) clean; full `dotnet test` green; build 0W/0E.
