## Approach (resolves the old Q1) — plain CSS, vendor + className
The design ships **plain, class-based CSS keyed to design tokens** — so the approach is settled and dependency-light:
1. **Vendor the tokens:** copy `colors_and_type.css` → `web/src/styles/tokens.css` (keep the Google-Fonts `@import` at top; it loads the fonts with no extra config).
2. **Vendor + adapt the styles:** copy `ui_kits/workbench/workbench.css` → `web/src/styles/workbench.css`; fix the token-import path; trim kit-only rules (`.toast-wrap` fixed positioning if we keep the banner inline, `DemoBar`, etc.). These class names (`.card`, `.btn`, `.seg`, `.pill`, …) become the classNames we add to components.
3. **Import once** in `web/src/main.tsx`: `import './styles/tokens.css'` then `import './styles/workbench.css'`.
4. **Add classNames + minimal wrappers** to `App.tsx` + each component, matching the kit's markup — **preserving every existing `aria-label`/`role`/button-text/`data-final`** (see Test-preservation below).

**No CSS Modules, no Tailwind** (the design is already global class-based; matching it 1:1 is simplest + transfers cleanly to H.2). **New deps expected + fine** (per the lead): Google Fonts (via the `@import`, zero dep) + **Lucide** (see design-Q1 below).

## ⭐ Store-field divergence map (the #1 risk — kit MOCK shape ≠ our real `UiSessionState`)
The implementer styles **our** components, which read **our** store. The kit's `s.*` are the mock's. Honor OUR shapes (display-only; no logic change):

| Concept | Kit MOCK reads | OUR real source (keep this) |
|---|---|---|
| Mode availability | `s.providerHealth.realtime === 'ready'` | `providerHealth: ConfigResponse` via the `modeAvailability()` / `availableModels()` **selectors** (ModeToggle/SessionSetup already call them) |
| `sessionStatus` | `idle/configured/starting/active/readyForTurn/ending/ended` | **same union** (kit only *display-remaps* `configured→config`, `readyForTurn→ready` for the pill var) |
| `turnStatus` | `ready/recording/processing/playing/completed/failed` | **same + `captured`** (ours has the extra `captured` state) |
| Top-level latency | `t.latency.speechEndToFirstAudioMs` direct | computed by **`deriveTurnMetrics(turn)`** over `turn.latencyEvents[]` (lesson §13 — the backend can't compute it for cascade). **Keep the `deriveTurnMetrics` call; do NOT switch to a direct `latency.X` read.** |
| Per-stage | `t.latency.stages` | `turn.latency.stages?: Record<string,number>` (same idea) |
| Cost | `t.cost.assumption` (singular), `t.estimatedCostPerMinuteUsd` | `turn.cost: CostEstimate` with `assumptions` (**array**) + `formatCostPerMinute()`/`formatUsdPerMinute()` helpers; per-mode from `state.summary.{cascade,realtime}.estimatedCostPerMinuteUsd` |
| Comparison data | `s.summary.byMode[mode]` + `s.summary.variants[]` + `s.summary.wer` | **different**: `ComparisonSummary` runs its own `loadComparison()` flow → `{summary:{realtime,cascade,wer,turnCount}, byVariant}`; the per-variant split is a frontend projection (lesson §21), NOT in `summary`. Style OUR `data` shape. |
| Errors | toast w/ `dismissError` per error | our `ErrorBanner` reads `state.errors`, renders `errorCopy(error)`, **has no dismiss** (adding dismiss = logic = out of scope — style only) |
| Transcript segments | `{text,isFinal}[]` | **same** (`sourceTranscript`/`targetTranscript` normalized by the store, lesson §10) — partial vs final styling maps directly |
| Brand mark | `../../assets/mark.svg` | copy `mark.svg` into `web/public/` (or `web/src/assets/`) + reference; it's a flagged placeholder (CAVEAT 1) — fine for H.1 |

## ⭐ Test-query preservation (the green-suite mechanic — load-bearing)
Component tests query by **role / `aria-label` / button text / `data-final`** — NOT by className. Adding classNames + wrapper `<div>`/`<span>`s is safe **iff** these are preserved. Must keep:
- **Button text**: `Cascade`/`Realtime`, `Start recording`/`Stop`, `Start session`/`End session`, `Record & evaluate`.
- **`aria-label`s**: `provider-health`, `mode-toggle`, `session-setup`, `recording-controls`, `turn-status`, `capture-timer`, `transcripts`, `source-unavailable`, `source-transcript`, `target-transcript`, `metrics`, `turn-top-level`, `turn-stages`, `session-averages`, `cost`, `turn-cost`, `session-cost`, `evaluation-panel`, `reference-text`, `wer-result`, `hypothesis`, `comparison-summary`, `Realtime-summary`, `Cascade-summary`, `wer-summary`, `cost-by-variant`, `error-banner`.
- **`role`s**: `alert` (ErrorBanner), `combobox` (selects), `button`.
- **`aria-pressed`** on ModeToggle buttons (we *style* the `aria-pressed="true"` state — never remove the attr).
- **`data-final`** on transcript `<li>`s.

**If a chosen markup restructure moves an `aria-label`-bearing element** (e.g. converting ComparisonSummary's `ModeColumn` divs to a `<table>`): you MAY adjust the **test query** (a query, not the assertion) and **note it in the write-up** — per lesson §14 the tests query structure, not behavior, so a query tweak that preserves the assertion is allowed. Prefer preserving the `aria-label` on a wrapping element where practical.

## Design questions to settle (Step-2.5-equivalent — answer in the styling write-up before/while applying)
1. **(dep) Lucide delivery.** The design standardizes on Lucide icons. Options: (a) **`lucide-react`** npm package (typed, tree-shakeable, idiomatic React 19 — `import { Mic } from 'lucide-react'`); (b) CDN `<script>` + `data-lucide` (the kit's vanilla approach — not idiomatic here). **Default vote: (a) `lucide-react`.** Use the README's canonical icon mapping (zap=Realtime, layers=Cascade, mic, square, gauge, dollar-sign, target, columns, alert-triangle, info, …). Confirm.
2. **ComparisonSummary structure.** Our component renders `ModeColumn` divs + a `cost-by-variant` list; the kit's look is a real `.cmp-table` (by-mode rows + indented variant rows). Options: (a) **convert to the `.cmp-table`** (matches the design centerpiece; may require moving the `Realtime-summary`/`Cascade-summary`/`cost-by-variant` `aria-label`s onto wrapping elements + a minimal test-query tweak — assertions unchanged); (b) keep the div structure, style as comparison cards in the mode color language. **Default vote: (a)** — the lead flagged the comparison grid as "the deliverable, most visual weight." Keep all `aria-label`s you can; note any query you adjust.
3. **ErrorBanner placement.** Kit uses a fixed top toast (`.toast-wrap`); ours is an inline `<section role="alert">`. Options: (a) **style inline as a `.toast`-styled error strip** (keep current placement + `role="alert"`, no positioning/logic change); (b) port the fixed top-center toast (positioning-only). **Default vote: (a)** — simplest, no layout/scroll surprises, zero behavior change. (No dismiss button either way — that's logic.)
4. **Markup-restructure bar.** Default: the **minimum** wrappers for layout/legibility (a card container, a controls row, a status pill, the transcript columns) — not a component rewrite. Confirm the bar: "CSS + the smallest markup that makes it match the kit + stays test-green," not "redesign the component tree."
