# /tdd brief — realtime_connection_lifecycle

## Feature
The realtime **connection lifecycle**: hoist the WebRTC connection to a persistent pc held across turns (replacing E.4b's per-turn lazy-connect), stamp the connection timing markers, surface an ICE disconnect (never swallow), and tear down cleanly on End Session. Flow-G mode-switch + re-mint/recovery are E.5b.

## Use case + traceability
- **Task ID:** E.5a (split from E.5 — "connection lifecycle, disconnect, recovery, mode-switch"; this slice is lifecycle + disconnect, E.5b is mode-switch + recovery).
- **Architecture sections it implements:** `ARCH-010` (§7 — Connection lifecycle + Failure handling), `ARCH-018` (errors surfaced, never swallowed), `ARCH-013` (the `realtime_connect_ms` metric).
- **Related context:**
  - E.4b shipped (`f2b5647`): `realtimeTurnController` with **lazy-connect** (connect on first `startTurn`); the client singleton + interim `<audio>` + once-stamped `playback.started`. E.5a hoists connect to a **persistent pc** + adds teardown.
  - E.3's `createRealtimeWebRtcClient` (the manual-smoke shell) owns the `RTCPeerConnection`. E.5a extends it (shell) with a settable raw `onConnectionState` delegate + `teardown()`; the connection-state **mapping** lives in a tested unit (the controller / a connection manager) — same shell-vs-tested split as E.4b (web §17).
  - The store actions (mode-agnostic): `appendLatencyEvent`, `failTurn(uiError)`, `addError(uiError)` (all shipped). `errorCopy(code)` map (web §14 / D.7) — E.5a adds the disconnect copy.
  - **Carry-forwards this slice discharges:** the failed-realtime-turn `Errors` population (origin B.9c-ii → E.5); the double-`connect()` orphan-leak guard (origin E.3 → E.5); the B.3-origin `realtime.session.connecting` stamp (moved from E.4b); the `<audio>` lifecycle teardown (origin E.4b → E.5).
  - ARCH-010 §7: "**One `RTCPeerConnection` is created at session start and held open across turns**; torn down only on End Session or mode-switch-away (so `realtime_connect_ms` is a one-time cost)." OpenAI caps a session at 60 min; the 5-min demo benchmark fits with **no mid-demo re-mint** → re-mint is E.5b / a documented fallback (NOT this slice).

## Acceptance criteria (what "done" means)
- [ ] **Persistent connect:** the connection is established once (at session-start / first turn) and **held across turns** — the turn controller no longer lazy-connects per turn; subsequent turns reuse the live pc. A 2nd connect while connected is a **no-op** (idempotent — discharges the double-`connect()` orphan-leak guard).
- [ ] **Connection timing stamps (browser clock):** `realtime.session.connecting` stamped at connect-**initiation** (the B.3-origin marker — so `realtime_connect_ms = connected − connecting` computes); `realtime.session.connected` on pc connectionstate `connected`.
- [ ] **Disconnect surfaced (never swallowed):** on pc connectionstate `failed`/`disconnected` → stamp `realtime.session.disconnected` + map to a sanitized `UiError` (code `realtime.session.disconnected`, `stage:'realtime'`) → `store.failTurn` (if a turn is active) / `store.addError`; **populate the failed turn's errors** (discharges the B.9c-ii → E.5 carry-forward). The error advises **switch-to-Cascade** (ARCH-010 default failure behavior).
- [ ] **`errorCopy` entry:** `errorCopy('realtime.session.disconnected')` → actionable copy ("Connection lost — switch to Cascade mode and retry." or similar), never blank/raw (web §14 sentinel discipline).
- [ ] **Teardown on End Session:** stop tracks, close the data channel, close the pc, release the `MediaStream`, **detach the `<audio>` `srcObject`**, and **reset the per-turn guards** (`playback.started` once-guard, the connected flag) — discharges the `<audio>`-lifecycle carry-forward.
- [ ] All unit tests pass; `/preflight` clean.

## RED test outline (Step 2)
1. **`connect_stamps_connecting_at_initiation`** — calling the persistent connect stamps `realtime.session.connecting` (browser clock) at initiation (before connectionstate fires). Why: B.3-origin / `realtime_connect_ms`.
2. **`connectionstate_connected_stamps_connected`** — `onConnectionState('connected')` → stamp `realtime.session.connected`. Why: ARCH-010 mapping.
3. **`persistent_connect_is_idempotent`** — connect when already connected → no-op (no 2nd pc/track; the controller reuses the live connection across turns). Why: ARCH-010 one-pc-held-across-turns + the double-`connect()` guard.
4. **`disconnect_stamps_and_fails_turn`** — `onConnectionState('failed')` (and `'disconnected'`) → stamp `realtime.session.disconnected` + `store.failTurn`/`addError` with `{code:'realtime.session.disconnected', stage:'realtime'}`; a planted raw string is absent (no-leak). Why: ARCH-010 failure handling + ARCH-018.
5. **`disconnect_populates_turn_errors`** — after a disconnect during an active turn, the turn carries the disconnect error (the B.9c-ii→E.5 failed-turn `Errors` discharge). Why: ARCH-005 turn errors.
6. **`errorCopy_disconnect_advises_switch`** — `errorCopy('realtime.session.disconnected')` returns actionable switch-to-Cascade copy, never blank/raw. Why: web §14 + ARCH-010 advise-switch.
7. **`teardown_closes_releases_and_resets`** — End → `teardown()` → (mocked) pc closed, DC closed, tracks stopped, stream released, `<audio>` `srcObject` detached, per-turn guards reset. Why: ARCH-010 teardown discipline + the `<audio>`-lifecycle discharge.

## Things to flag at Step 2.5
1. **Where does the connection-state mapping live — extend `realtimeTurnController` or a new `realtimeConnectionManager`?** My default vote: **a small `realtimeConnectionManager` (or a section of the controller)** that owns connect/teardown/connection-state→store — keeps the turn controller focused on turns. Flag your preference based on the E.4b controller's shape.
2. **Persistent connect trigger — session-start vs first-turn?** ARCH-010 says "created at session start." My default vote: **connect when the realtime session becomes active** (session-start for realtime mode) so `realtime_connect_ms` is a one-time pre-first-turn cost; if the existing flow makes first-turn cleaner, first-turn is acceptable (still held after). Flag the trigger.
3. **Disconnect during no-active-turn → `addError` only (not `failTurn`)?** My default vote: **yes** — `failTurn` only if a turn is active; otherwise `addError` (a session-level disconnect). Flag if the store distinguishes.
4. **Advise-switch mechanism — `errorCopy` copy only, or also a store flag?** My default vote: **`errorCopy` copy only** (the actionable message tells the user to switch; an auto-switch is out of scope / E.5b). Flag if a store "suggestedMode" signal is cleaner.
5. **Re-mint on `expiresAt` — confirm DEFERRED.** ARCH-010: the 60-min cap > the 5-min demo, so no mid-demo re-mint. My default: **defer re-mint to E.5b / a documented fallback** (NOT this slice). Flag if you think a minimal `isExpired` guard belongs here.
