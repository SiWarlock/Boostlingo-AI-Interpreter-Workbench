## Feature
Enhance the `SessionHistory` panel (067): the previous-sessions list lives in a **fixed-height scrollable container**
(internal scroll, no unbounded growth); clicking a session row **expands it in place** (accordion) to render that
session's **summary aggregates** (turnCount + cascade/realtime avg per-stage latencies, estimated cost, error count,
WER) AND a **per-turn breakdown** (each turn's per-stage latencies, transcripts, status, cost, model used).

## Design (FE consumption + presentation; reuse existing renderers)
- **Part 1 ‚Äî bounded scroll:** wrap the session list in a fixed max-height container with `overflow-y:auto` (a
  `.hist-list`/`.hist-scroll` cap ‚Äî confirm the height approach, Q4). The list scrolls internally; the panel doesn't
  grow unbounded.
- **Part 2 ‚Äî accordion:**
  - `sessionsApi.getSession(id)` ‚Üí `GET /api/sessions/{id}` (¬ß3; a non-OK/`session.not_found` ‚Üí `ApiError(UiError)`).
  - A DI'd `historyActions.loadSessionDetail(id)` (¬ß7) ‚Üí `getSession(id)` ‚Üí returns the detail; a failure routes a
    sanitized `UiError` (reuse `sessions.read_failed` or a `session.detail_failed` copy ‚Äî Q-minor) to the store sink.
  - `SessionHistory`: each row is a clickable accordion header; clicking toggles expand. On first expand, fetch the
    detail ONCE and cache it (transient local state keyed by `sessionId`, ¬ß20-style ‚Äî re-expand doesn't refetch; a
    past session is immutable, Q2). Single-open (Q1). Render inline, below the row:
    - **Summary aggregates** from the embedded `summary` (turnCount + the cascade/realtime `ModeSummary` avg per-stage
      latencies + estimated cost + errorCount + `WerSummary`). Reuse the MetricsPanel summary presentation where clean.
      Handle an absent/null `summary` gracefully (a "summary unavailable" note ‚Äî a session that never computed one).
    - **Per-turn breakdown** from `turns[]` ‚Äî each turn: mode chip (blue/violet), status, per-stage latencies
      (`deriveTurnMetrics` over its `latencyEvents`, ¬ß25), transcripts (source/target), cost (`costEstimate`, ¬ß21),
      `translationModelUsed`, WER (if `werResult`/`isEvaluation`). Reuse the per-turn metrics/cost renderers where they
      aren't coupled to the live store (Q3).
  - A detail-fetch failure ‚Üí the sanitized banner (store sink, ¬ß2); the row shows an inline error/empty, doesn't crash.

## Acceptance (RED-first; deterministic ‚Äî jsdom ¬ß14 + fetch-mock ¬ß4; the scroll cap is presentational)
- [ ] **Bounded scroll:** the list renders inside a fixed-height container with `overflow-y:auto` (assert the
      constraint is applied ‚Äî the container has the cap class/style; the actual scroll is visual/manual).
- [ ] **`sessionsApi.getSession`** GETs `/api/sessions/{id}` ‚Üí the full session; a non-OK ‚Üí `ApiError(UiError)` (¬ß3).
- [ ] **`historyActions.loadSessionDetail`** returns the detail on success; routes a sanitized `UiError` to the store
      + returns null on failure (both arms; no raw leak ‚Äî ¬ß7/¬ß2).
- [ ] **Row click expands + fetches once:** clicking a row dispatches `loadSessionDetail` √ó1, renders the summary
      aggregates + the per-turn breakdown; re-collapse/re-expand does NOT refetch (cached, Q2). Clicking another row
      collapses the first (single-open, Q1).
- [ ] **Per-turn breakdown renders** each turn's mode/status/stage-latencies/transcripts/cost/model from `turns[]`
      (jsdom ‚Äî query by role/text; pin N turns, the order, a cascade + a realtime turn if both present).
- [ ] **Summary renders** the aggregates; an absent `summary` ‚Üí a graceful "unavailable" note (not a crash).
- [ ] **Detail-fetch error** ‚Üí the sanitized banner via the store sink (never raw); the row doesn't crash.
- [ ] `/preflight` (web) clean.
