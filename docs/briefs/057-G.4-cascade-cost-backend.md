## Context / root cause (grounded; confirm exact lines at Step 1)
**(a)** The per-turn anchor lives in **`Metrics/MetricsAggregator.cs` → `Compute(IReadOnlyList<LatencyEvent>)`** — it is **mode-agnostic** (takes only events):
- line ~26 `recordingStopped = Get(LatencyEventNames.TurnRecordingStopped)`
- lines ~33–36 `firstOutputAudio = Get(TtsFirstAudio) ?? realtimeFirstAudioDelta ?? playbackStarted`
- line ~41 `SpeechEndToFirstAudioMs = Between(recordingStopped, firstOutputAudio)`
- line ~48 `Get(LatencyEventNames.SttFinal)` **already exists** in this method.
`SessionSummaryService.cs` (~line 52/56) reuses `MetricsAggregator.Compute` per-turn then averages `m.SpeechEndToFirstAudioMs` into `ModeSummary.AvgSpeechEndToFirstAudioMs`. **Realtime emits NO `stt.final`** (ARCH-013 ~line 1063 MUST-list: realtime = connected/recording.stopped/first_audio_delta/playback.started/completed) → so a universal `Get(SttFinal) ?? recordingStopped` anchor fixes cascade and **leaves realtime unchanged via the fallback** — no mode-branch needed (confirm at Step 1 that no realtime path emits `stt.final`).

**(b)** Degrade chain:
- `Providers/OpenAI/OpenAiTranslationMapping.cs` → `ReadUsage` (~lines 101–113) reads usage **nested under `response`**: `root.response.usage.{input_tokens, output_tokens}` (snake_case) and **returns `(null,null)` if the `response.usage` object is absent** (or either token missing).
- `Cascade/CascadeStreamingOrchestrator.cs` → `StampTranslationFinal` (~lines 419–433) writes `inputTokens`/`outputTokens` into the `translation.final` event **Metadata**.
- `Cascade/CascadeWsMapping.cs` → `FoldCostInputs` (~lines 55–85) sums those Metadata tokens across segments.
- `Cost/CostEstimator.cs` → `EstimateTranslation` (~lines 56–74; `usage.InputTokens ?? 0`) + `EstimateCascadeTurn` (~lines 204–240) — **wholesale degrade at ~212–220: if ANY stage `!IsSuccess` → `Unavailable(...)`**.
- `Cascade/CascadeWebSocketEndpoint.cs` → `EmitTerminalAsync` (~lines 177–200) only sends a `cost{estimate}` frame when `ToCostMessageOrNull` is non-null → a degraded estimate sends **no** frame ("No priced turns yet").
- **There is NO raw-usage logging today** → we're blind to the live shape. The prime suspect: the live OpenAI Responses streaming-terminal `usage` is NOT at `response.usage.{input_tokens,output_tokens}` (different nesting/field names) → `ReadUsage` reads `(null,null)` → translation stage `Unavailable` → composite wholesale-degrades → no cost frame.

**(c)** `Cascade/CascadeStreamingOrchestrator.cs` → `SynthesizeSegmentAsync` (~lines 325–410): `tts.started` is stamped on the provider's **`TtsStarted`** event (~line 387); `tts.first_audio` on the provider's dedicated **`TtsFirstAudio`** sentinel (~lines 390–392). They are stamped on **distinct provider events** — so 0 ms means the real OpenAI TTS provider (C.3) yields `TtsStarted` then `TtsFirstAudio` within the same `_clock.UtcNow` tick (provider-synchronous-yield / clock-resolution / buffering artifact), **not** a stamping bug. No speculative fix.
