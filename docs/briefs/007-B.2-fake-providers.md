# /tdd brief — fake_providers

## Feature
Implement the three fake providers — `FakeSttProvider`, `FakeTranslationProvider`, `FakeTtsProvider` — each implementing its B.1 interface, with configurable **variants** emitting ordered events, a **configurable per-event delay** (via `yield return` + awaited delay), and honoring `CancellationToken`, per ARCH-012. These are the deterministic test doubles every later Phase-B/C seam exercises (and they let the host run keyless locally via DI swap).

## Use case + traceability
- **Task ID:** B.2
- **Architecture sections it implements:** `ARCH-012` ("Fake providers (test doubles)" — variants + configurable delay + cancellation), `ARCH-020` (testing strategy — deterministic seams).
- **Related context:** Implements the B.1 interfaces (`ISttProvider`/`ITranslationProvider`/`ITtsProvider`, events, requests, `AudioFrame`) — all landed (e65e13e). Consumed by **B.4** (cascade orchestrator tests), **B.10** (the comprehensive `ProviderBoundaryTests`), and the host (DI swaps fakes↔real by config). Live in `Providers/Fakes/` (the **Api project**, not the test project) so they're injectable.

## Acceptance criteria (what "done" means)
- [ ] **`FakeSttProvider`** variants per ARCH-012: **success-with-partials** (`SttStarted → SttPartial* → SttFinal`), **empty-final** (`SttStarted → SttFinal("")` — feeds the `cascade.empty_transcript` path), **partials-then-error** (`SttStarted → SttPartial* → SttFailed(ProviderError)`).
- [ ] **`FakeTranslationProvider`** variants: **token-stream-then-final** (`TranslationStarted → TranslationPartial(delta)* → TranslationFinal(text, tokens)`), **immediate-final-only** (`TranslationStarted → TranslationFinal` — no partials), **error** (`TranslationStarted → TranslationFailed`).
- [ ] **`FakeTtsProvider`** variants: **chunked-then-complete** (`TtsStarted → TtsFirstAudio → TtsAudioChunk* → TtsComplete`), **complete-only** (`TtsStarted → TtsFirstAudio → TtsComplete` — no chunks beyond first), **error** (`TtsStarted → TtsFailed`).
- [ ] Each emits via `async IAsyncEnumerable<*Event>` with `[EnumeratorCancellation]`, a **configurable delay before each event** (default fast/0 for tests; settable so B.4 can drive timing scenarios), and **honors `CancellationToken`** — cancelling mid-stream stops emission + surfaces `OperationCanceledException` (the awaited delay observes the token).
- [ ] Tests verify, per fake + variant: the **ordered event sequence**; **cancellation mid-stream → `OperationCanceledException`**; the failure variants emit a `*Failed(ProviderError)` (using the B.1 `ProviderErrorMapper` / a fake `ProviderError`). Delay *timing* is NOT wall-clock-asserted (just that it's configurable + doesn't reorder events).
- [ ] `/preflight` clean (`dotnet build` 0W/0E + `dotnet test` + `dotnet format`).
- [ ] Cross-doc invariant: **none** (fakes are test doubles, not contracts).

## RED test outline (Step 2 — `FakeProvidersTests.cs`)
Per fake, drive each variant by enumerating the `IAsyncEnumerable` and collecting events:
1. **`fake_stt_success_emits_started_partials_final`** — ordered `SttStarted, SttPartial+, SttFinal`. Why: ARCH-012.
2. **`fake_stt_empty_final`** — `SttStarted, SttFinal("")` (empty text; no partials required). Why: feeds cascade empty-transcript.
3. **`fake_stt_partials_then_error`** — ends in `SttFailed` after ≥1 partial. Why: partial-failure path (B.4).
4. **`fake_translation_token_stream_then_final`** — `TranslationStarted, TranslationPartial(delta)+, TranslationFinal`. Why: ARCH-012 streaming.
5. **`fake_translation_immediate_final_only`** — `TranslationStarted, TranslationFinal` (no partials). Why: ARCH-012 variant.
6. **`fake_translation_error`** — `TranslationStarted, TranslationFailed`. Why: ARCH-012.
7. **`fake_tts_chunked_then_complete`** — `TtsStarted, TtsFirstAudio, TtsAudioChunk+ (Seq ordered), TtsComplete`. Why: ARCH-012 streaming.
8. **`fake_tts_complete_only`** — `TtsStarted, TtsFirstAudio, TtsComplete` (no extra chunks). Why: ARCH-012 variant.
9. **`fake_tts_error`** — `TtsStarted, TtsFailed`. Why: ARCH-012.
10. **`cancellation_mid_stream_throws_oce`** — (one fake, representative) cancel the token after the first event → enumeration throws `OperationCanceledException`; no further events. Why: ARCH-012 cancellation contract (B.4 relies on it for timeouts).

## Things to flag at Step 2.5
1. **Test file.** My default vote: focused **`FakeProvidersTests.cs`** (B.2-scoped fake contracts). **B.10 NEW-creates `ProviderBoundaryTests.cs`** (the CRITICAL cross-cutting suite — interface contracts + mapper integration + cancellation across all three). Keeps B.2's fake-unit tests separate from B.10's boundary integration. (Alt: seed `ProviderBoundaryTests` now — but two clear files is cleaner.)
2. **Variant selection mechanism.** My default vote: a **constructor parameter** (an `enum` per fake, e.g. `FakeSttBehavior.SuccessWithPartials | EmptyFinal | PartialsThenError`) + a `delayPerEvent` (default `TimeSpan.Zero`) + optional scripted payloads (the partial/final texts, chunk count). Simple, deterministic, lets B.4 configure timing. Flag if you'd prefer a config record.
3. **Delay + cancellation mechanics.** My default vote: `await Task.Delay(delay, ct)` before each `yield return` — this both applies the delay AND observes cancellation (throws `OperationCanceledException` cleanly), satisfying both requirements in one place. Default delay `Zero` so tests are fast.
4. **Failure-variant `ProviderError`.** My default vote: build it via the B.1 `ProviderErrorMapper` (e.g. `Map(new HttpRequestException(...), provider, stage)`) or a direct `ProviderError` literal — either is fine for a fake; reuse the mapper if it reads cleanly. Confirm.
5. **`SttFinal("")` for empty-final.** My default vote: empty string (not null) — `cascade.empty_transcript` is triggered by an empty/whitespace final text in B.4, so the fake emits `SttFinal("")`. Confirm.
