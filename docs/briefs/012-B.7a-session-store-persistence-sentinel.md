# /tdd brief — session_store_and_persistence_writer (B.7a) — SAFETY

## Feature
`SessionStore` (in-memory session/turn state) + `SessionPersistenceWriter` (write-on-end MUST + best-effort per-turn; `session_YYYYMMDDTHHMMSSZ_<short-id>.json`; **path-traversal guard**; **never persists secrets or raw audio**). The persistence **sentinel test is the safety centerpiece** of this slice.

## Acceptance criteria (what "done" means)
- [ ] `SessionStore`: create a session (server-generated `sessionId` matching `^[A-Za-z0-9_-]+$`), get by id, add/update turns, record mode transitions, end a session. Thread-safe for concurrent HTTP + WS access (→ 2.5 Q1).
- [ ] `SessionPersistenceWriter.WriteAsync(session)`: serializes via `JsonDefaults` to `SESSION_DATA_DIR/session_<UTC yyyyMMddTHHmmssZ>_<short-id>.json`; returns the path (Flow F shows it). **MUST** on session end; **best-effort** per completed/failed turn (→ 2.5 Q4).
- [ ] **Path-traversal guard (#5):** a `sessionId` containing `../`, `/`, `\`, `.`, or any char outside `^[A-Za-z0-9_-]+$` is **rejected** (no file written outside `SESSION_DATA_DIR`); the resolved full path is canonicalized and asserted to stay under `SESSION_DATA_DIR` (defense-in-depth — two layers, → 2.5 Q3).
- [ ] **Sentinel (safety centerpiece — #1/#2/#3):** the serialized session JSON contains **no standard API key, no `ek_` ephemeral secret, and no raw audio bytes**, while still containing transcripts + latency events (round-trips). (→ 2.5 Q2 for the assertion strategy.)
- [ ] Round-trip: write → read back → the deserialized session equals the original (assert on JSON-string / field values, not record `==` — lesson §2).
- [ ] Persistence IO failure degrades to a `persistence.failed` normalized error (ARCH-018), never an unhandled crash (per-turn best-effort; write-on-end surfaces success/failure).
- [ ] All unit tests in `AiInterpreter.Tests/SessionPersistenceTests.cs` pass.
- [ ] `/preflight` clean. **Mandatory `security-reviewer` pass at Step 8** (invariant-touching slice).

## RED test outline (Step 2) — `AiInterpreter.Tests/SessionPersistenceTests.cs`
The **sentinel + path-traversal are the safety pins** — write them first, make them strong.

1. **`round_trip_preserves_session`** — build a session with config + a turn (transcripts + latency events + cost + a normalized error) → `WriteAsync` → read the file → deserialize → assert field-level equality (transcripts/latency/cost present); assert via JSON-string/values, not record `==` (lesson §2).
2. **`sentinel_json_excludes_api_key`** — set a sentinel standard key (e.g. config holds `sk-SENTINEL-DO-NOT-PERSIST`) → write a session → the file text does **NOT** contain the sentinel key substring. (#1)
3. **`sentinel_json_excludes_ephemeral_secret`** — a sentinel `ek_SENTINEL...` value present in adjacent runtime state → the file text does **NOT** contain `ek_` / the sentinel. (#2)
4. **`sentinel_json_excludes_raw_audio`** — even if a turn is built from cascade output that carried audio, the persisted JSON contains **no base64 audio blob / no `TtsAudioChunk.Bytes`-shaped data**; structurally the model has no audio field — assert no audio bytes leak. (#3)
5. **`path_traversal_sessionid_rejected`** — `sessionId = "../../etc/passwd"` (and `"a/b"`, `"a.b"`, `"a\\b"`) → `WriteAsync` rejects (throws/`Result.Failure`), **no file created outside `SESSION_DATA_DIR`**. (#5)
6. **`resolved_path_stays_under_data_dir`** — a server-generated id → the canonicalized resolved path `StartsWith(SESSION_DATA_DIR full path)` (defense-in-depth layer 2). (#5)
7. **`io_failure_degrades_to_persistence_failed`** — unwritable dir / IO error → `persistence.failed` (or `Result.Failure`), no unhandled exception. (ARCH-018)
8. **`store_create_generates_valid_id`** — `SessionStore.Create(...)` → `sessionId` matches `^[A-Za-z0-9_-]+$`; get-by-id returns it; turns add/update.
9. **`store_concurrent_turn_adds`** *(if Q1 warrants)* — concurrent turn appends don't corrupt the turn list / lose turns.

## Things to flag at Step 2.5
1. **`SessionStore` concurrency model.** HTTP (B.9) + WS (C.4) touch a session concurrently. My default vote: **`ConcurrentDictionary<string, InterpretationSession>` + a per-session lock object** for turn-list/transition mutations (the record is immutable but holds a mutable `List<>`). Simple + correct for MVP. Alternative: copy-on-write replace of the whole session record — more allocation, no lock; flag if you prefer it.
2. **Sentinel assertion strategy.** My default vote: **inject sentinel values** (a fake standard key in config + an `ek_SENTINEL` in adjacent runtime state) and assert the serialized JSON text does NOT contain them, **plus** the structural fact (the model has no key/secret/audio field). The sentinel-string scan is the active, drift-proof defense (catches a future field that leaks); structural alone is too passive. Assert transcripts/latency ARE present (so we're not trivially-empty-passing).
3. **Path-traversal guard — layers.** My default vote: **both** — (a) reject any `sessionId` not matching `^[A-Za-z0-9_-]+$` (fast, before touching the FS); (b) canonicalize (`Path.GetFullPath`) the resolved path and assert it `StartsWith` the `SESSION_DATA_DIR` full path (defense-in-depth). Layer (a) alone is sufficient but (b) guards against a future id-format change.
4. **Write tiers.** My default vote: **write-on-end is MUST** (awaited, surfaces success/failure for Flow F); **per-turn is best-effort** (filtered catch → log + `persistence.failed`, never throws into the turn/stream flow). Atomic temp→rename durability is a documented TRIM (ARCH-025) — NOT in this slice.
5. **DI registration scope.** My default vote: **register `SessionStore` (singleton) + `SessionPersistenceWriter` + bind `SESSION_DATA_DIR` now** (consumer B.9). Consistent with B.3/B.5. Flag if you'd rather defer to B.9.
6. **`SESSION_DATA_DIR` binding + dir creation.** My default vote: a small options/config read (env `SESSION_DATA_DIR`, default `data/sessions`), and the writer ensures the directory exists (create-if-absent) on first write. Confirm `.env.example` already lists `SESSION_DATA_DIR` (A.2).
7. **`sessionId` / filename format.** My default vote: `sessionId = "session_" + <short-id>` where `<short-id>` is a lowercase alphanumeric GUID segment (matches `^[A-Za-z0-9_-]+$`); filename `session_<UTC yyyyMMddTHHmmssZ>_<short-id>.json` (timestamp from `IClock`). The optional label is a JSON field only — **never in the filename** (ARCH-016).
