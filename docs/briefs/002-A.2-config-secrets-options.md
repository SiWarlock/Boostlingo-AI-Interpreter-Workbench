## Things to flag at Step 2.5
1. **`record` vs `class` for Options (IOptions binding semantics).** IOptions binding needs **settable properties + a parameterless ctor** — positional records don't bind. ARCH-005 says *domain models* are immutable records, but Options are a config-binding concern, not domain models. My default vote: **`class` with `{ get; set; }`** (or `record` with `{ get; init; }` + a parameterless ctor) and default values inline — the standard bindable pattern. Don't force immutable positional records here.
2. **Flat-env-var → config-section mapping.** ARCH-028 env vars are flat screaming-snake (`DEEPGRAM_API_KEY`); Options bind from sections (`Deepgram:ApiKey`). My default vote: **A.2 binds from PascalCase sections** (tests use in-memory sections); `.env.example` documents the flat operator-facing names; the explicit env→section remap (or `Deepgram__ApiKey` convention) is wired in **A.5/Program.cs**. Don't build the bridge in A.2.
3. **PricingOptions scope.** Minimal binding target now vs full ARCH-014 shape. My default vote: **minimal now** (`version` + forward-compatible), A.4 extends — keeps A.2 decoupled from ARCH-014.
4. **`appsettings` secret-absence test (#6).** My default vote: **include it** — cheap ARCH-019 defense-in-depth; the heavier secret-exclusion pins are persistence (B.7) + config endpoint (B.9).
5. **OpenAiTtsOptions per-language voice map.** My default vote: **include the optional nullable map property now** (it's in ARCH-012 enumerated); leave it unused until C/E.
