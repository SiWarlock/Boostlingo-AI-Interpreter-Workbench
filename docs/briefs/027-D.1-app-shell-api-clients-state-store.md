# /tdd brief ‚Äî app_shell_api_clients_state_store

## Feature
Stand up the Phase-D frontend foundation in `web/`: the TS contract-mirror types, a typed `sessionStore` holding `UiSessionState` (ARCH-007), the three D.1 API clients (`configApi`/`sessionsApi`/`cascadeApi`) over a shared `http` helper, and the React+Vite app shell that bootstraps `GET /api/config` into the store on mount. `VITE_API_BASE_URL` + a Vite dev proxy wire the SPA to the backend API + cascade WS.

### Verified backend contract shapes (mirror these EXACTLY ‚Äî confirmed against `server/` source at HEAD)
> These are the on-disk record shapes; the TS types are camelCase projections (backend serializes camelCase + enum-as-camelCase-string + explicit-null via `JsonDefaults`, server lesson ¬ß2). **Where this differs from an ARCH-009 doc example, the source wins.**

- **`GET /api/config` ‚Üí `ConfigResponse`** (`Config/ConfigModels.cs`):
  `{ realtime: { configured: bool, models: string[] }, cascade: { stt: { configured, provider, model }, translation: { configured, provider, models: string[] }, tts: { configured, provider, model } }, languages: string[], pricingConfigVersion: string }`
- **`POST /api/sessions` ‚Üí full `InterpretationSession`** (`Sessions/SessionModels.cs`) ‚Äî top-level id field is **`sessionId`** (NOT `id`; the abbreviated ARCH-009 example is correct on this). Shape the store consumes:
  `{ sessionId: string, label?: string, startedAt: string, endedAt?: string, config: { currentMode: 'realtime'|'cascade', direction: { source, target }, providerProfile: { realtimeModel, translationModel, ttsVoice, sttModel, ... } }, turns: [...], modeTransitions: [...], summary?: ..., pricingConfigVersion: string }`
  ‚Üí request `CreateSessionRequest`: `{ label?: string, mode: 'realtime'|'cascade', direction: { source, target }, realtimeModel: string, translationModel: string }`.
- **`POST /api/sessions/{id}/turns` ‚Üí `CreateTurnResponse`**: `{ turnId: string }`.
- **`POST /api/sessions/{id}/end` ‚Üí `EndSessionResponse`**: `{ session: InterpretationSession, persistedPath?: string, persistenceWarning?: UiError }` (exactly one of path/warning non-null; **200**, never 500).
- **`POST /api/cascade/turn` (multipart) ‚Üí `CascadeTurnResponse`** (`Cascade/CascadeBlobModels.cs`): `{ turn: InterpretationTurn, audioBase64?: string, audioContentType?: string, persistenceWarning?: UiError }`. **Form fields** (`CascadeTurnForm`): `sessionId`, `turnId?`, **`source`** + **`target`** (separate `LanguageCode` fields ‚Äî NOT a nested `direction` object), `translationModel`, `ttsVoice`, `audio` (file).
- **Enums** (camelCase-as-string): `InterpretationMode`‚Üí`'realtime'|'cascade'`; `LanguageCode`‚Üí`'en'|'es'`; `SessionStatus`‚Üí`'idle'|'configured'|'starting'|'active'|'readyForTurn'|'ending'|'ended'`; `TurnStatus`‚Üí`'ready'|'recording'|'captured'|'processing'|'playing'|'completed'|'failed'`. These are exactly the ARCH-007 `UiSessionState` union values.
- **Error bodies:** B.9 routed-path errors return a sanitized **`UiError`** `{ code, safeMessage, stage?, retryable, turnId? }` (e.g. `session.not_found` 404). Framework validation (400) + unrouted 404/405 return ProblemDetails (server lesson ¬ß14) ‚Äî a different shape; the `http` helper must tolerate both.

## Acceptance criteria (what "done" means)
- [ ] `web/src/types/domain.ts` defines the enums/unions (`InterpretationMode`, `LanguageCode`, `SessionStatus`, `TurnStatus`, `LatencyStage`, `ClockSource`), `LanguageDirection`, the **view models** `UiSessionState`/`TurnViewModel`/`UiError` (verbatim ARCH-007 ¬ß4 shapes), and the D.1 **wire DTOs** the clients parse (`ConfigResponse` + nested capability types, `CreateSessionRequest`, the consumed `InterpretationSession` shape, `CreateTurnResponse`, `EndSessionResponse`, `CascadeTurnResponse`). `tsc --noEmit` clean; no `any` on exported surfaces.
- [ ] `sessionStore` (`web/src/state/sessionStore.ts`) exposes a `createSessionStore()` factory returning `{ getState, subscribe, ...actions }` over an immutable `UiSessionState`; the initial state matches ARCH-007 defaults; each action produces a **new** state object (reference changes) and notifies subscribers; `subscribe` returns an unsubscribe fn. The store OWNS the `sessionStatus`/`turnStatus` state machine (these are NOT on the wire).
- [ ] `configApi.getConfig()` GETs `${base}/api/config` ‚Üí `ConfigResponse`. `sessionsApi.{createSession, getSession, createTurn, endSession}` hit their documented routes with correct method/body and parse the typed response. `cascadeApi.postCascadeTurn(params, audioBlob)` POSTs `multipart/form-data` to `/api/cascade/turn` ‚Üí `CascadeTurnResponse`, **always sending `source` + `target`** (mitigates the C.5 `[Required]` gap client-side).
- [ ] Shared `http` helper builds URLs from `import.meta.env.VITE_API_BASE_URL ?? ''`, performs the fetch, and on a non-OK response maps the body ‚Üí a typed `ApiError` carrying a `UiError` (parses a real `UiError` body when shape matches; synthesizes a generic `UiError` `{ code: 'http.<status>', ... }` otherwise). It **never** surfaces a raw response body / stack to callers.
- [ ] App shell (`web/src/App.tsx`) renders from the store and, on mount, calls `configApi.getConfig()` ‚Üí `store.loadConfig(...)` (Flow A bootstrap); a fetch failure dispatches `store.addError(...)` (no unhandled rejection). Renders current `sessionStatus` + a basic provider-health/error readout. **No transport-client internals imported into the shell** (clean separation, ARCH-007 / forbidden-pattern #3).
- [ ] `vite.config.ts` adds a dev proxy for `/api` (with `ws: true` for the cascade WS) so `VITE_API_BASE_URL=''` works zero-config in dev; `test.environment` switches to `jsdom` only if a render test lands here (else stays `node` ‚Äî see Step-2.5 Q on whether any component test is in D.1; default: no, shell is manual-smoke).
- [ ] All unit tests in `web/src/state/*.test.ts` + `web/src/api/*.test.ts` pass (Vitest).
- [ ] `/preflight` clean (`npm run lint && npm run typecheck && npm run test`).
- [ ] Cross-doc: the first `web/src/types/` models land ‚Üí flag at Step 9 so the orchestrator populates the `web/CLAUDE.md` cross-doc invariants table (currently "none yet").

## Things to flag at Step 2.5
1. **TS-mirror completeness ‚Äî full domain vs pragmatic-accrete?** Mirror the ENTIRE Appendix A domain now, or only the fields D.1's clients/store consume (typing `InterpretationSession.turns`/`modeTransitions`/`summary` loosely with a "consumed in D.4/D.6/F" marker)? My default vote: **pragmatic-accrete** ‚Äî mirror only what D.1 reads; defer the turn/summary nested wire types to the slice that first consumes them. Matches the per-slice mirroring discipline in `web/CLAUDE.md` + avoids over-building the foundation. (If you prefer full-now for completeness, say so ‚Äî both are defensible; full-now means more cross-doc rows in one round.)
2. **Client error model ‚Äî throw typed `ApiError` vs return a `Result`-union?** Default vote: **throw `ApiError extends Error { uiError: UiError }`** ‚Äî idiomatic for React (callers `try/catch` ‚Üí `store.addError(e.uiError)`); a Result-union adds ceremony the UI doesn't need here. The store stays the single error sink.
3. **Vite dev wiring ‚Äî proxy vs direct+CORS?** Default vote: **dev proxy for `/api` (`ws:true`) + clients read `VITE_API_BASE_URL ?? ''`** so dev is zero-config (base `''`), while a full `VITE_API_BASE_URL` + the already-configured backend CORS/Origin enables direct mode. Either way the TDD'd client code is base-agnostic (only `vite.config.ts`/env differ). Note: the cascade WS Origin check (C.4b) needs `FRONTEND_ORIGIN` = the browser-presented origin (`http://localhost:5173`) ‚Äî true for both proxy + direct.
4. **Default `mode` in initial state?** Default vote: **`'cascade'`** ‚Äî Phase D builds the cascade UI first; Realtime is Phase E; config-gating (D.2) disables an unconfigured mode regardless. (Minor; the ARCH-009 create example uses `realtime` but that's illustrative.)
5. **`sessionStarted` target status ‚Äî `'active'` vs `'readyForTurn'`?** Default vote: **`'active'`** on create, with `'readyForTurn'` reserved for after the first turn-setup completes (D.2/D.4 refine). Keeps D.1's mapping simple; revisit when the turn lifecycle lands.
6. **`sessionsApi` surface ‚Äî include `completeTurn`/`appendEvents` now?** Default vote: **no** ‚Äî those are the REALTIME turn-finalize paths (Phase E); D.1 ships `createSession`/`getSession`/`createTurn`/`endSession` (the Phase-D-relevant set). Avoids extra tested-but-unwired surface.

## Lessons-logged candidates anticipated (‚Üí `web/LESSONS.md` ‚Äî frontend sequence, NOT `server/`)
