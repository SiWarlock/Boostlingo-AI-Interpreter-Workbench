# /tdd brief — cascade_blob_fallback (C.5)

## Feature
The documented **non-streaming** cascade fallback: `POST /api/cascade/turn` (multipart) takes an uploaded audio blob → **pre-recorded** STT → the SAME streamed translation→TTS pipeline → a single JSON turn response (transcripts + latency + cost + status, persisted). Includes **audio-upload validation** (size cap + content-type allowlist → `cascade.invalid_audio` 413/415 — **key-safety-rule #5**, its own commit), the ARCH-020 provider-boundary suite extended with real-provider/HTTP-mock cases, a **SafeMessage sentinel sweep** across the real-provider suites, and the 0-byte-audio TTS contract pin. **C.5 closes Phase C.**

## Use case + traceability
- **Task ID:** C.5 (closes Phase C).
- **Architecture sections it implements:** `ARCH-009` (§6 — `POST /api/cascade/turn` shape + the blob response), `ARCH-011` (§8 — the pre-recorded fallback is the documented non-streaming path; reuse the streamed translation/TTS), `ARCH-008` (§5 — **provider/orchestration logic lives in a service, NEVER the controller**), `ARCH-019` (§15 — audio-upload validation = safety invariant #5), `ARCH-018` (§15 — normalized error codes + degrade), `ARCH-020` (§15 — the interface-level boundary suite + the real-provider sweep, lesson §17).
- **Related context:**
  - **STT pre-recorded routing already exists (C.1).** `DeepgramSttProvider.TranscribeAsync` routes by `request.Encoding`: `linear16` → live WS; **any other (container) encoding → the REST pre-recorded path** (`DrainAsync` the `AudioFrame`s → single blob → REST → a single `SttFinal`). So the blob path needs no new provider code — feed the uploaded bytes as a single `AudioFrame` with the **container** encoding (e.g. `webm`/`ogg`/`wav`) and the existing provider does pre-recorded STT.
  - **Reuse the streaming pipeline, don't re-implement it.** `CascadeStreamingOrchestrator.RunAsync(p, audioFrames, ct)` already does STT → per-segment translation→TTS → `Done`, emits the flat `IAsyncEnumerable<CascadeOutputEvent>`, and (post-C.4b) **fails closed** on terminal-less streams. The blob path feeds it a 1-frame audio stream + a container-encoding `CascadeStartParams`, then **collects** the event stream into a turn result + cost — reusing `CascadeWsMapping.AssembleTurn` + the cost fold + `CostEstimator.EstimateCascadeTurn` (the same glue the WS endpoint uses, minus the socket). See Step-2.5 Q1.
  - **`CascadeController` does NOT exist yet** — C.5 creates it. The dir has `ConfigController`/`SessionsController`. Controller stays **thin** (validate upload, delegate to the service, map `Result`→DTO; lesson §15/§16).
  - **C.6 already resolved the 429 reconciliation.** A real Deepgram 429/401/403 now maps to `rate_limited`/`auth` via the `err_code`→`MapStatus` recovery (C.6), so the boundary-suite 429 case works **as written**; only a Deepgram **5xx-with-body** still degrades to `stt.unknown` (the accepted C.6 residual — assert that honestly, don't fight it).
  - **Distinct validation boundaries (no conflict with C.4a/C.4b):** the WS `encoding` allowlist is `{linear16,pcm}` (raw PCM only); the **blob content-type allowlist** is the containers (`audio/webm`/`ogg`/`wav`/`mpeg`/`mp4`, ARCH-019 item 8). Different surfaces — the blob path does NOT go through `CascadeStartValidation`.
  - **Reuses:** the C.4b `CascadeWsMapping.AssembleTurn`/cost fold + the C.4b `ProviderErrorMapper.Unknown` (fail-closed comes free), B.5 `CostEstimator`, B.7 `SessionStore`/`SessionPersistenceWriter`, B.10 `ProviderBoundaryTests` (lesson §17 — interface-level), the B.7a persistence-sentinel pattern (for the new SafeMessage sweep). Lessons §8, §17, §21, §22.

- [ ] **security-reviewer pass** (invariant #5 — uploads size- + content-type-validated; this commit is its surface).

- [ ] `POST /api/cascade/turn` (multipart) accepts the audio blob + the turn params (`sessionId`/`turnId`/`direction`/`translationModel`/`ttsVoice`; the **encoding is derived from the content-type**, not client-supplied); validates the turn exists; runs the pre-recorded cascade; returns a single JSON turn response (shape per Q2); persists best-effort (ARCH-016).

## Things to flag at Step 2.5
1. **Reuse `CascadeStreamingOrchestrator` vs a separate blob orchestrator.** My default vote: **reuse it** — `CascadeOrchestrator` is a thin adapter (blob → 1-frame `IAsyncEnumerable<AudioFrame>` with the container encoding → `RunAsync` → collect events via `CascadeWsMapping.AssembleTurn` + the cost fold). Maximal reuse, the pre-recorded STT + per-segment + fail-closed + cost all come free; ARCH-008/011, lesson §8/§21. Reject a re-implementation (divergence risk). Confirm the adapter shape + that feeding a single `AudioFrame` with a container encoding correctly hits the REST path.
2. **Blob response shape.** The single JSON response: the full assembled `InterpretationTurn` (transcripts + latency + cost + status) — and does it include the **target TTS audio** (base64) since it's not streamed, or transcripts+metrics only? My default vote: **return the assembled turn (transcripts/latency/cost/status) + the concatenated target audio as base64** (the client needs audio to play; the WS path streamed it, the blob path must deliver it in-body) — confirm against ARCH-009's blob-response shape, and confirm the audio is in the **response only**, never persisted (invariant #3).
3. **0-byte-audio TTS contract (origin C.3).** Today: empty body → `TtsStarted → TtsComplete` (no `TtsFirstAudio`). My default vote: **keep as-is + pin it** (real TTS always returns audio; the MVP contract tolerates `FirstAudio`-absent — the §17 boundary suite already keeps success assertions tolerant of the no-chunk variant). Only switch to `TtsFirstAudio`-always / `TtsFailed` if you see a real need. Confirm.
4. **Cost-trail polish (carry-forward, last-consumer C.5 — origin B.5).** (a) `CostEstimate.Units` always writes `cachedAudioInputSeconds` (0 when unsupplied) → omit the key when null so the trail distinguishes "no cache" from "0s"; (b) `EstimateTranslation` substitutes 0 for null tokens (silent $0) vs STT's degrade-on-null. My default vote: **document both as intentional for the MVP + leave the code** (omitting-when-null is a JSON-shape change touching the cost cross-doc; not worth a Phase-C-close detour) — OR a tiny align-to-degrade for (b) if it's a one-liner. Your call; don't silently drop — decide + note.
5. **Commit count.** My default vote: **2–3.** **(1) upload validation — SAFETY, own commit** (invariant #5, non-negotiable isolation + security-reviewer pass).
