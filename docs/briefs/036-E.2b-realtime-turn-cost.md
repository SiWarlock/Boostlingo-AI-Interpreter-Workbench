# /tdd brief — realtime_turn_cost

## Feature
Wire realtime per-turn cost estimation: on `POST …/turns/{turnId}/complete` for a **Realtime**-mode turn, compute the turn's `CostEstimate` via the existing `CostEstimator.EstimateRealtime(...)` (currently unwired — zero callers), so a realtime turn carries a cost for the cross-mode comparison.

- **Architecture sections it implements:** `ARCH-014` (cost estimation — realtime audio-seconds→tokens basis), `ARCH-009` (`/complete` contract), `ARCH-013` (metrics/turn shape).
- **Related context:**
  - `CostEstimator.EstimateRealtime(string model, CostUsage usage, long audioDurationMs)` (`Cost/CostEstimator.cs:151`) is **already implemented + unit-tested** (lesson §9) — this slice only **wires** it; do not re-implement the math. It branches on the model (`gpt-realtime`/`gpt-realtime-mini`), reads `usage.AudioInputSeconds`/`AudioOutputSeconds`/`CachedAudioInputSeconds`, applies `RealtimeTokensPerAudioSecond=50`, and **degrades to `Unavailable`** (pricing absent / rates absent) via `Result<CostEstimate>`.

  - The realtime model lives on **`SessionConfig.RealtimeModel`** (`SessionModels.cs:33`; set at `CreateSession`). The turn carries `Mode` (`InterpretationMode.Realtime`/`Cascade`), `AudioDurationMs`, and a currently-null `CostEstimate?` (`SessionModels.cs:68-82`).

It already routes through the **idempotent** `_store.FinalizeTurn` transform (C.4b — an already-terminal turn is returned unchanged). `SessionService` currently injects `_pricing` but **not** `CostEstimator` — this slice adds the `CostEstimator` dependency.
  - **Pattern to mirror:** how the cascade WS path sets a turn's cost (`CascadeWsMapping` fold → `EstimateCascade`); E.2b is the realtime analog at `/complete`. Lessons **§9** (branch-on-basis, degrade via `Result`, `0.0`-rate ≠ unavailable), **§12** (propagate null-semantics; never 0/throw).

## Things to flag at Step 2.5
1. **⭐ Output audio-seconds sourcing (the central fork).** Realtime bills input AND output audio at different rates; `CompleteTurnRequest` carries only `audioDurationMs` (input). Options: **(a)** add an optional `OutputAudioDurationMs?` to `CompleteTurnRequest` — forward-built for E.4 to report (the frontend plays the output audio, so it can measure it); absent → output disclosed-unavailable. **(b)** estimate output ≈ input × a disclosed factor (no contract change, less accurate). **(c)** input-only now, output deferred to E.4. **My default vote: (a)** — the cost comparison is a deliverable and omitting output would badly skew realtime-vs-cascade; the optional field is a light, honest, forward-compatible contract addition (mirrors B.9c-ii forward-building `/events`). Absent → `Assumptions` discloses "output audio not counted," never a synthetic 0. Accept the small cross-doc change.
2. **`CostEstimator` injection into `SessionService`.** Default: inject the existing singleton `CostEstimator` (don't re-derive cost from `_pricing` by hand). Vote: **inject `CostEstimator`**.
3. **Where to compute.** Inside the `FinalizeTurn` transform (atomic with terminal status, idempotent) vs after. Vote: **inside the transform** — co-located with the other turn-finalize fields; idempotent by construction.
4. **Degrade target.** On `EstimateRealtime → Unavailable`, leave `CostEstimate` null (honest; summary tolerates it) vs store an "unavailable" marker. Vote: **null** (lesson §12 null-semantics; no 0, no marker record).
5. **Input seconds source.** `request.AudioDurationMs` vs the already-merged `turn.AudioDurationMs` (the transform sets `AudioDurationMs = request.AudioDurationMs ?? turn.AudioDurationMs`). Vote: **use the merged turn value** so a turn that accrued duration elsewhere is respected.
