# /tdd brief — session_summary_service (B.7b)

## Feature
`SessionSummaryService` — a **pure, read-only** aggregation service that computes a `SessionSummary` from an `InterpretationSession` on demand: per-mode `ModeSummary` (averaged latency metrics + averaged cost/min + error count), a session-level `WerSummary`, and turn counts. No new model fields; no persistence; no mutation.

> **B.7 is split into two slices for safety-commit isolation:** B.7a (store + writer + sentinel — SAFETY, landed `9bfea76`) and **this brief (B.7b — `SessionSummaryService`, read-only aggregation, separate commit).** This is the non-safety completion of B.7.

## Acceptance criteria (what "done" means)
- [ ] `SessionSummaryService.Compute(InterpretationSession) → SessionSummary` — **pure** (returns a new `SessionSummary`; does NOT mutate the session or write `session.Summary`). The caller decides whether to store the snapshot (B.9 `/end`).
- [ ] `SessionSummary.TurnCount` = total turn count; `ComputedAt` = injected `IClock` value; `PricingConfigVersion` = the session's.
- [ ] `Realtime` / `Cascade` `ModeSummary` is populated **only** for a mode that has ≥ 1 turn; **`null`** when that mode has no turns (→ 2.5 Q3). Each `ModeSummary.TurnCount` counts only that mode's turns.
- [ ] Per-mode averaged metrics come from `MetricsAggregator.Compute` per turn, then mean over **non-null** values (all-null → `null`, never `0` or error — → 2.5 Q3): `AvgSpeechEndToFirstAudioMs`, `AvgSpeechEndToPlaybackMs` (universal); `AvgSttFinalMs`, `AvgTranslationFinalMs`, `AvgTtsFirstAudioMs` (cascade-only — **`null` for realtime turns**, which emit no such events).
- [ ] `ModeSummary.EstimatedCostPerMinuteUsd` = average of the mode's non-null per-turn `CostEstimate.EstimatedUsdPerMinute` (`null` when none priced) (→ 2.5 Q1).
- [ ] `ModeSummary.ErrorCount` = sum of `turn.Errors.Count` across the mode's turns (→ 2.5 Q2).
- [ ] `WerSummary` (session-level, both modes): `SampleCount` = turns with a non-null `WerResult`; `AvgWer` = mean of those `Wer` values (**unbounded — no clamp**, lesson §10); `null` when no turn has a `WerResult` (→ 2.5 Q6).
- [ ] Empty session (0 turns) → `SessionSummary(TurnCount=0, Realtime=null, Cascade=null, Wer=null, ComputedAt, PricingConfigVersion)` — no throw.
- [ ] All unit tests in `AiInterpreter.Tests/SessionSummaryServiceTests.cs` pass. `/preflight` clean.

## RED test outline (Step 2) — `AiInterpreter.Tests/SessionSummaryServiceTests.cs`
Use a deterministic `IClock` (fixed `ComputedAt`) + the real `MetricsAggregator` (compose the seam, don't mock it — same pattern the metrics tests use). Build turns with literal ARCH-013 wire-name `LatencyEvent`s so the aggregator produces known `TurnMetrics`.

1. **`empty_session_returns_zero_summary`** — 0 turns → `TurnCount=0`, all `ModeSummary`/`Wer` null, `ComputedAt` from clock, `PricingConfigVersion` echoed. No throw.
2. **`single_cascade_turn_populates_cascade_summary`** — one cascade turn (full stt/translation/tts + universal events + cost + 1 error) → `Cascade` populated (TurnCount=1, avg metrics = that turn's values, cost/min, ErrorCount=1); `Realtime=null`.
3. **`single_realtime_turn_cascade_fields_null`** — one realtime turn (universal events only) → `Realtime` populated with `AvgSttFinalMs`/`AvgTranslationFinalMs`/`AvgTtsFirstAudioMs` = **null**; universal avgs present; `Cascade=null`. (ARCH-005 line 326.)
4. **`multi_turn_averages_metrics`** — two cascade turns with different latency values → each `Avg*` = arithmetic mean of the two. Pins the averaging math.
5. **`mixed_mode_session_splits_summaries`** — a session with both realtime + cascade turns → both `ModeSummary`s populated, each counting only its mode; top-level `TurnCount` = total.
6. **`null_metric_excluded_from_average`** — turns where a metric is present on some, null on others → mean over present only; a metric null on **all** turns → `null` (n/a, never `0`/throw).
7. **`error_count_sums_per_mode`** — turns with varying `Errors.Count` → `ModeSummary.ErrorCount` = sum for that mode (not count-of-failed-turns).
8. **`wer_summary_aggregates_all_turns_unbounded`** — turns with `WerResult` (some null, one with `Wer > 1.0`) → `SampleCount` = count present, `AvgWer` = mean **including the >1.0 sample (no clamp)**; turns without WER excluded; no WER anywhere → `Wer=null`.
9. **`cost_per_minute_averaged_over_non_null`** — turns with/without `CostEstimate.EstimatedUsdPerMinute` → average of non-null; all-null → `null`.
10. **`computed_at_and_version_from_inputs`** — `ComputedAt` == injected clock value; `PricingConfigVersion` == the session's (staleness disambiguation per ARCH-009).

## Things to flag at Step 2.5
1. **Cost/min aggregation.** `ModeSummary.EstimatedCostPerMinuteUsd` = **average of the mode's non-null per-turn `EstimatedUsdPerMinute`** (each turn's value is already a per-minute projection; the summary is a labeled estimate-of-estimates). Alternative: blended `sum(EstimatedUsd) / sum(turn-minutes from AudioDurationMs)` — duration-weighted, arguably "truer," but mixes models within a mode. My default vote: **average the per-turn rates** (simpler, consistent with the other `Avg*` fields, decimal? null when none). Flag if you prefer blended.
2. **`ErrorCount` semantics.** My default vote: **sum of `turn.Errors.Count`** across the mode's turns (total normalized errors), not count-of-failed-turns. Matches "error counts" in the ARCH-009/ARCH-007 comparison intent.
3. **Null-averaging rule.** My default vote: average over **non-null** values only; a metric absent on every turn → `null` (n/a), never `0` or throw — mirrors `MetricsAggregator`/`TurnMetrics` "missing → null, never error" (lesson §7). A `ModeSummary` exists iff its mode has ≥ 1 turn (an empty mode → the whole `ModeSummary` is null, not a zero-filled one).
4. **Model-variant breakdown (acceptance-criteria boundary).** The final-submission criterion "summary compares modes **and** model variants (both realtime + both translation models)" — the current `SessionSummary`/`ModeSummary` records are **mode-level only** (one cost/min per mode). My default vote: **B.7b fills the existing mode-level records (no field change); the model-variant grouping is F.3's job**, computed client-side from the raw turn list (each `InterpretationTurn` carries `TranslationModelUsed`; realtime model is on `ProviderProfile`). Extending the summary record for server-side per-model breakdown would be a **cross-doc invariant change → escalation** — defer unless F.3 proves it needs server aggregation. Confirm you agree, or flag F.3-needs-it now.
5. **Pure compute vs write-through.** My default vote: **`Compute` is pure** — returns a `SessionSummary`, does NOT set `session.Summary`. The cached-snapshot write on `/end` (ARCH-009) is B.9's persistence step (compute → store into the session record → persist). Keeps the service stateless + trivially testable. Flag if you'd rather it write through.
6. **`WerSummary` scope.** My default vote: **session-level, across both modes** (the record is not per-mode). `SampleCount`/`AvgWer` over every turn with a non-null `WerResult`; **never clamp `Wer > 1.0`** (lesson §10).
7. **DI registration.** My default vote: **register `SessionSummaryService` now** (singleton, stateless; consumer B.9) — consistent with B.3/B.5/B.7a. Flag if you'd defer to B.9.
