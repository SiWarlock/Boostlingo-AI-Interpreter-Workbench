## Acceptance criteria (what "done" means)
- [ ] **Normalization:** lowercase (invariant) → strip punctuation → collapse whitespace; **accents preserved** (ARCH-015: accent-stripping only if explicitly documented — default preserves language text). `NormalizedReference`/`NormalizedHypothesis` on the result reflect this.
- [ ] **WER:** `WerCalculator.Compute(phraseId, reference, hypothesis) → WerResult` with `Wer = (Substitutions + Insertions + Deletions) / ReferenceWordCount`, `ReferenceWordCount = N` = normalized reference word count.
- [ ] **S/I/D attributed individually** via DP edit-distance backtrace (not just a total distance) — the record stores each.
- [ ] **Perfect match → `Wer == 0`** (S=I=D=0).
- [ ] **One deletion → D=1, S=I=0, `Wer == 1/N`**; **one insertion → I=1**; **one substitution → S=1**.
- [ ] **Empty hypothesis → all reference words deleted → D=N, `Wer == 1.0`**.
- [ ] **Empty reference (N=0 after normalization) → explicitly rejected, no divide-by-zero** (mechanism → 2.5 Q1).
- [ ] **Punctuation/casing normalization:** `"Hello, World!"` vs `"hello world"` → `Wer == 0`.
- [ ] `EvaluationPhraseStore` loads `evaluation-phrases.json` → 8–12 phrases, **both EN and ES present**; lookup by `phraseId` + filter by `Language`. Load failure (missing/malformed) degrades (→ 2.5 Q2), never crashes.
- [ ] `evaluation-phrases.json` created with 8–12 EN+ES phrases (interpretation-relevant categories: appointment, intake, etc.).
- [ ] All unit tests in `AiInterpreter.Tests/WerCalculatorTests.cs` pass.
- [ ] `/preflight` clean.

## RED test outline (Step 2) — `AiInterpreter.Tests/WerCalculatorTests.cs`
CRITICAL tier (ARCH-020) — the ARCH-015 case list is the spec; cover it fully.

**`WerCalculator`:**
1. **`perfect_match_is_zero`** — identical ref/hyp → S=I=D=0, `Wer==0`.
2. **`one_deletion`** — hyp drops one word → D=1, S=0, I=0, `Wer==1.0/N`.
3. **`one_insertion`** — hyp adds one word → I=1, `Wer==1.0/N`.
4. **`one_substitution`** — hyp swaps one word → S=1, `Wer==1.0/N`.
5. **`empty_hypothesis_all_deleted`** — hyp `""` → D=N, S=I=0, `Wer==1.0`.
6. **`empty_reference_rejected_no_divide_by_zero`** — ref `""` (or punctuation-only → 0 normalized words) → rejected per 2.5 Q1 (throw `ArgumentException` / `Result.Failure`), NOT `NaN`/`Infinity`/throw-DivideByZero.
7. **`punctuation_and_casing_normalized`** — `"Hello, World!"` vs `"hello world"` → `Wer==0`; `NormalizedReference=="hello world"`.
8. **`accents_preserved_in_normalization`** — `"Necesito Ayuda."` vs `"necesito ayuda"` → `Wer==0` and the `ñ`/accented forms in a Spanish phrase are NOT stripped (e.g. `"añode"` stays `año`-bearing). Pins the accent-preserve decision (2.5 Q4).
9. **`sid_attribution_combined`** — a hyp with one sub + one insertion + one deletion vs a multi-word ref → S=1,I=1,D=1, `Wer==3.0/N` (pins individual attribution + the tie-break of 2.5 Q3).

**`EvaluationPhraseStore`:**
10. **`loads_phrases_en_and_es_present`** — load the committed file → 8–12 phrases; ≥1 `LanguageCode.En` + ≥1 `LanguageCode.Es`; `GetById` returns the right phrase; `GetByLanguage(Es)` filters.
11. **`missing_or_malformed_file_degrades`** — bad path / malformed JSON → degrade per 2.5 Q2 (empty list / `Result.Failure`), no crash.

## Things to flag at Step 2.5
1. **Empty-reference (N=0) rejection mechanism.** My default vote: **throw `ArgumentException`** — the reference is always a known, validated scripted phrase (from the store), so an empty reference is a precondition violation/bug, not a degradable runtime state; fail-fast surfaces it (F.1's controller maps it to a 400). Alternative: `Result<WerResult>.Failure`. (Either satisfies "no divide-by-zero"; pick the one that reads as "this shouldn't happen" vs "this can happen and we degrade.")
2. **Phrase-store load-failure handling.** My default vote: **degrade like `PricingLoader`** (lesson §3 — missing-file guard + size guard + filtered catch → `Result<IReadOnlyList<EvaluationPhrase>>` or an empty store), so a missing file shows "phrases unavailable" rather than crashing the host. The eval panel is a committed MUST but shouldn't take down the app.
3. **DP S/I/D tie-break.** Equal-cost edit paths can attribute operations differently. My default vote: **standard WER convention — on an equal-cost cell prefer substitution (diagonal) over insertion/deletion**, and document the precedence so the count is reproducible. The simple test cases (2–5) are unambiguous; test 9 pins the combined case under the chosen tie-break.
4. **Accent handling — confirm preserve.** My default vote: **preserve accents** (ARCH-015 default); normalization = invariant-lowercase + strip `\p{P}` punctuation (incl. `¿¡`) + collapse whitespace, leaving accented letters intact. (Accent-stripping is a documented opt-in we are NOT taking.)
5. **DI registration scope.** My default vote: **register `EvaluationPhraseStore` (startup singleton, loads once) + `WerCalculator` now**, consumer = F.1 — consistent with B.3/B.5. Alternative: defer to F.1. (If `WerCalculator` is pure/stateless you may prefer a static — but an injectable service is more testable + consistent; your call, flag if static.)
6. **Phrase-file path.** My default vote: a configurable `EVALUATION_PHRASES_PATH` defaulting to the `Evaluation/evaluation-phrases.json` content location (copied to output via csproj), loaded through the same degrade path. Confirm the `CopyToOutputDirectory` wiring works for tests.
