## Feature
Backend ephemeral-credential broker: `POST /api/realtime/client-secret` mints a short-lived OpenAI Realtime client secret (`ek_…`) by calling the GA `client_secrets` endpoint with the standard key server-side, and returns **only** the ephemeral secret + expiry + model to the browser.

## Acceptance criteria (what "done" means)

**Happy path**
- [ ] `POST /api/realtime/client-secret` with `{ sessionId, direction:{source,target}, model? }` returns `200` + `{ clientSecret:"ek_…", expiresAt:"<ISO-8601>", model:"<resolved>" }` (ARCH-009 §6 shape).
- [ ] The upstream call is **`POST https://api.openai.com/v1/realtime/client_secrets`** (GA) with `Authorization: Bearer <RealtimeOptions.ApiKey>` and a body matching ARCH-010: `expires_after:{anchor:"created_at", seconds:<ExpirySeconds>}`, `session:{type:"realtime", model, instructions, output_modalities:["audio"], audio:{input:{turn_detection:null, transcription:{model:"<TranscriptionModel>"}}, output:{voice:"<Voice>"}}}`. An `OpenAI-Safety-Identifier` header is set.
- [ ] **Never `/v1/realtime/sessions`** — pinned by a request-target assertion.
- [ ] Upstream `client_secret.value → clientSecret`; `client_secret.expires_at` (Unix epoch int) → `expiresAt` ISO-8601 (`DateTimeOffset.FromUnixTimeSeconds(...).UtcDateTime`-style); `model` echoed.

**SAFETY (invariant pins — load-bearing)**
- [ ] **#1 — standard key never crosses the boundary:** with `RealtimeOptions.ApiKey = "sk-SENTINEL…"` and a forced upstream failure (mock 401), the **entire serialized HTTP response** (and the `UiError.safeMessage`) contains no `sk-` substring and no raw upstream payload/stack. Sentinel-scan test (mirror the persistence sentinel posture).
- [ ] **#2 — ephemeral secret never persisted/leaked:** the minted `ek_…` appears **only** in the success `RealtimeTokenResponse.clientSecret`. The broker performs **no persistence call** (structurally — no `SessionStore`/`SessionPersistenceWriter` on the path) and does **not** log the secret value. Pinned by: a success test asserting the `ek_` is returned, + the `security-reviewer` confirming no secret/key logging.

**Error path (sanitized, ARCH-018)**
- [ ] Upstream 401/403 → `realtime.auth` `UiError`, non-retryable, status surfaced. 400/422 → `realtime.invalid_request`. 5xx / network → `realtime.upstream_unavailable`. Timeout (OCE, caller-ct not set) → `realtime.timeout`. Each via `ProviderErrorMapper` + `ToUiError` (no bespoke codes).
- [ ] **One bounded auto-retry on 429** honoring `Retry-After`; a second 429 → sanitized `realtime.rate_limited` after **exactly one** retry (no infinite loop). The retry delay is test-controllable (no real wall-clock sleep in tests).

**Wiring / boundary / quality**
- [ ] **Reachable from** the real route `POST /api/realtime/client-secret` (`RealtimeController` → `RealtimeClientSecretService`), DI-wired in `Program.cs` (typed `HttpClient`, `BaseAddress = https://api.openai.com/`) — invoked on the real path, not just from tests (Step 7.5).
- [ ] Request strings capped (`sessionId`, `model` ≤ 256, §16); `model` resolved/validated (see Step-2.5 Q4).
- [ ] All new unit + transport + controller-wire tests pass; backend suite stays green (284 → 284 + new).
- [ ] `/preflight` clean (`dotnet format --verify-no-changes && dotnet build && dotnet test`).
- [ ] **Own commit** (SAFETY slice); `security-reviewer` pass logged at Step 7→8.

## Things to flag at Step 2.5
1. **Pure-mapper split.** Follow §18/§20 — extract a pure `internal static RealtimeClientSecretMapping` (body-build / instructions-render / response-parse / `ToFailed`) and keep the service a thin transport shell, OR keep it inline in the service. My default vote: **split** — it's the proven C.2/C.3 pattern and makes the body/parse logic unit-TDD'able without a network round-trip.
2. **Service → controller failure contract.** How does the service hand a failure to the controller? Options: (a) return a small outcome record carrying `RealtimeTokenResponse?` + `ProviderError?`; (b) `Result<RealtimeTokenResponse>` with the `ProviderError` surfaced. My default vote: **(a) a tiny outcome record + controller maps via `_sanitizer.ToUiError(err)` + `StatusCode(err.HttpStatusCode ?? 502, …)`** — mirrors `CascadeController` exactly, keeps code/stage/retryable/status intact to the boundary.
3. **The single bounded retry.** Which statuses retry, and how is the delay bounded + tested? My default vote: **429 only, exactly one retry**, honoring `Retry-After` (seconds form; absent/unparseable → a small fixed backoff; wait capped at `TokenTimeoutSeconds`); 5xx + network **fail fast** as `realtime.upstream_unavailable`. The delay must be **test-injectable** (or `Retry-After: 0` in tests) — no real `Task.Delay` wall-clock in the suite (lesson §6 determinism).
4. **`model` resolution + allowlist.** The request carries `model?`. My default vote: **resolve = request model if present else `RealtimeOptions.Model`; allowlist the resolved model against the known realtime catalog `{gpt-realtime, gpt-realtime-mini}`** (source: `ConfigService` catalog or a `RealtimeOptions`-derived set); off-catalog → `400 realtime.invalid_request`; cap `model`/`sessionId` ≤ 256. This **discharges the realtime-model portion of the model-allowlist carry-forward** (origin B.9c-i) at the mint boundary — note it at Step 9 so I can update that carry-forward item.
5. **Missing standard key.** If `RealtimeOptions.ApiKey` is null/empty. My default vote: **short-circuit to a sanitized `realtime.auth` (or `realtime.unavailable`) with no doomed upstream call** — mirrors the capability-from-key-presence posture (lesson §15). Register the broker DI unconditionally (it reads the key at call time).
6. **`OpenAI-Safety-Identifier` value.** My default vote: a **fixed server-side constant** (e.g. `"ai-interpreter-workbench"`) — it's an abuse-tracking id, not a secret; no config surface needed for the MVP.
